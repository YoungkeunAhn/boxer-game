# TASK-010 전용 스킬 슬롯·액티브/패시브 (v1.3d)

## 목표

타입 전용 스킬 시스템을 도입한다. **기본 4종(고정) + 액티브 슬롯(최대 3) + 패시브 슬롯(1)** 구조로 장착하고, 인파이터/아웃복서 전용 스킬을 구현해 앞선 메커닉(그로기·카운터·회피 버프·다단 타격)에 연결한다.

## 참고 문서

- `docs/기획/skills/equip.md`, `docs/기획/skills/infighter-skills.md`, `docs/기획/skills/out-boxer-skills.md`
- `docs/기획/boxer/infighter.md`, `docs/기획/boxer/out-boxer.md`

## 작업 범위

1. 스킬 데이터: `src/data/skills.ts`(신규)
   - 스킬 정의 타입: `id`, `name`, `type`(공유/인파이터/아웃복서), `kind`(`ACTIVE | PASSIVE`), 효과 메타, 쿨타임.
   - 인파이터: 리버샷(내상+그로기), 철벽가드(패시브 피해감소), 압박(훅/어퍼 버프+몬스터 약화), 가젤펀치(단일 강타+그로기), 뎀프시롤(다단 훅+대량 그로기).
   - 아웃복서: 고스트스텝(완전 회피+카운터 강화), 나비 스텝(회피율/쿨타임/카운터 버프), 스텝백 카운터(패시브 자동 반격), 팬텀 잽(다단 잽), 거리 조절(몬스터 쿨 지연+회피 버프).
   - 가정: 모든 데미지·쿨타임·버프 수치는 미확정이므로 `가정:` 임시값. 일부 문서 명시값(지속 5초/6초, +15%/+20%/+10% 등)은 그대로 반영.
2. `src/game/types.ts`
   - `Boxer`에 `equippedSkills: { active: SkillId[]; passive: SkillId | null }`(기본 4종은 고정이라 비저장 가능) 추가.
   - `CombatRuntime`에 `skillCooldowns: Record<SkillId, number>`, 활성 버프 상태(회피율 가산·쿨 가속·몬스터 약화 등) 추가.
   - 저장 형태 변경 → `SCHEMA_VERSION` 상향.
3. `src/game/skills.ts`(신규, 순수): 액티브 스킬 발동 판정(가정: 쿨타임이 끝나면 자동 발동, 발동 정책 명시), 효과 적용(데미지·그로기·내상 DoT·다단 타격·버프 부여), 패시브 상시 적용.
4. `src/game/combat.ts`: 스킬 효과를 전투 루프에 연결 — 그로기 누적 소스(리버샷·뎀프시롤·가젤펀치), 회피율/카운터 버프(고스트스텝·나비스텝·스텝백 카운터), 피해감소(철벽가드), 몬스터 쿨 지연(거리 조절·압박), 다단 타격(뎀프시롤·팬텀잽), 내상 DoT.
5. `src/game/constants.ts`: 가정 수치 모음 + 슬롯 수(액티브 최대 3, 패시브 1). `BALANCE_VERSION` 상향.
6. 장착 UI 골격(`gameStore` 액션 + 컴포넌트): 타입에 맞는 전용 스킬만 슬롯에 장착/해제. 상세 UI는 TASK-011에서 정리.
7. 테스트: 스킬별 효과(그로기/카운터/회피/피해감소/다단/DoT), 쿨타임, 타입 제약(인파이터 스킬은 인파이터만), 패시브 상시 적용, 장착/해제.

## 구현 원칙

- 스킬 효과는 순수 함수(`skills.ts`)로 계산하고 `combat.ts`가 호출한다. 전투 메커닉(그로기·카운터·회피)은 앞 태스크가 만든 지점에 "소스/버프"로 연결만 한다.
- 자동 발동 정책(쿨 종료 시 자동, 우선순위)은 `가정:`으로 명시하고 한 곳에서 관리.

## 하지 않을 것

- 스킬 해금 조건·획득 경제(가정: 초기엔 타입별 기본 제공, 해금은 TODO).
- 스킬 연출/텍스트(TASK-012). 여기서는 발동 결과를 상태로 노출만.

## 완료 기준

- 타입에 맞는 액티브 3 + 패시브 1을 장착하고, 전투 중 쿨타임에 따라 효과가 적용된다.
- 인파이터 스킬은 그로기/내상/피해감소, 아웃복서 스킬은 회피/카운터에 실제로 작용한다(결정적 테스트).
- 타입에 맞지 않는 스킬은 장착되지 않는다.
- `node tools/check.mjs full` 통과.
