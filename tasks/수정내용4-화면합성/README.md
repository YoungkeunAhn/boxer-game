# 구현 태스크 (수정내용4: 파이터 메인화면 — 최종 화면 합성)

`수정내용3`(TASK-014~018)이 파이터 메인화면의 각 기능을 **독립 프레젠테이셔널 컴포넌트**로 추가하는 데까지 끝낸 상태를 기준선으로 삼는다.
이 폴더는 그 컴포넌트들을 목업(`docs/ui/예시이미지/메인ui1.png`)의 **단일 게임 화면으로 합성·재배치**하는 후속 작업을 다룬다.

> 왜 별도 폴더인가: 수정내용3의 TASK-014~018은 "기능을 컴포넌트로 추가"가 목표였고 완료됐다. 합성·레이아웃은 어느 태스크에도 없어 미구현으로 남았고(`docs/ui/02-파이터-메인화면.md` §5 갭, 수정내용3의 TASK-025로 스펙만 존재), 성격이 다르므로(표현·합성 전용) 구현 묶음을 여기로 분리한다.

## 진행 순서

| 태스크 | 파트 | 제목 | 핵심 산출물 |

| [TASK-026](./TASK-026-fighter-screen-composition.md) | 02 | 파이터 메인화면 최종 합성 | `CombatStage` 단일 무대, `BoxerStatus`/`SkillPanel` 정리, 목업 수직 재배치 (표시 전용·아트 교체 제외) |

> 본 묶음은 수정내용3 `02-파이터-메인화면/TASK-025`(합성 스펙)를 **구현 단위로 구체화**한 것이다. 기획 근거·중복 진단·testid 보존 목록은 `docs/ui/02-파이터-메인화면.md` §5와 TASK-025 문서를 그대로 따른다.

## 범위 경계 (이 묶음에서 지키는 선)

- **표시 전용 합성만**: `src/game/`·`formulas`·`constants`·스토어 액션 무변경. `BALANCE_VERSION`/`SCHEMA_VERSION` 불변(저장 키 `boxer-game.save.v8` 유지).
- **아트는 플레이스홀더 유지**: `BoxerFigure`(🧍/👊)·`CombatPanel`(🥊/👾) 이모지 그대로. 실제 픽셀 스프라이트/링 배경 교체는 **별도 태스크**(에셋 확정 후).
- **순수 함수·주입 원칙**: 컴포넌트는 스토어 셀렉터/액션만 사용. `Date.now`/`Math.random`/타이머 직접 사용 금지.
- **MVP 경계**: 신규 게임 기능·강화 축·탭 추가 없음. 월드맵 동작은 여전히 `TODO`(비활성 표기 유지).

## 공통 규칙 (CLAUDE.md / AGENTS.md)

- **검증 게이트**: 변경 후 `node tools/check.mjs fast "<바뀐 파일>"`. UI/CSS/스토어 변경이므로 `npm run e2e`를 **반드시** 추가로 돌린다(이 묶음의 최대 리스크 = `data-testid` 보존).
- **버전 관리**: 표시 전용이라 범프 없음. 만약 합성 중 저장 형태나 수식을 건드리면 즉시 중단하고 범위를 재검토한다(이 묶음에서는 건드리지 않는다).
- **테스트**: 새 순수 함수 추가가 없도록 한다(합성은 JSX/CSS 재배치). 불가피하게 셀렉터 파생 로직이 생기면 colocated `*.test.ts`로 검증.
