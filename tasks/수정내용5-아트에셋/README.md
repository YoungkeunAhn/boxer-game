# 구현 태스크 (수정내용5: 아트 에셋 교체 — 이모지 → 실제 이미지)

`수정내용4`(TASK-026)가 파이터 메인화면을 **단일 `CombatStage`로 합성**하면서 아트는 **플레이스홀더(이모지) 유지**로 남겨둔 상태를 기준선으로 삼는다. 이 폴더는 그 플레이스홀더를 **실제 이미지 에셋으로 교체**하는 후속 작업을 다룬다.

> 왜 별도 폴더인가: TASK-026은 "표시 전용 합성"이 목표였고 아트 교체를 명시적으로 범위 밖(`하지 않을 것`)으로 뒀다. 합성이 만든 `data-animation-key`/`data-pose`/`data-boxer-type`/`data-gender` 슬롯에 에셋을 끼우는 작업은 성격이 다르고 에셋 확정에 의존하므로 여기로 분리한다.

## 진행 순서

| 태스크 | 제목 | 핵심 산출물 |
| --- | --- | --- |
| [TASK-027](./TASK-027-art-assets.md) | 아트 에셋 교체 | 스프라이트 에셋 명세, 파이터 정지 이미지 연결(idle 완료), 포즈 시트·몬스터·링 배경 끼울 자리 |

## 착수 기준선 (이미 된 것 / TASK-026 후속으로 처리)

- 파이터 4종(타입×성별) **정지 idle 이미지**는 `public/sprites/`에 넣고 `BoxerFigure`가 이모지 대신 렌더하도록 **연결 완료**(이 세션에서 처리). 이모지는 에셋 누락 시 폴백으로만 남음.
- 아직 이모지/텍스트로 남은 것: **포즈별 스프라이트 시트**(공격·가드·회피·카운터), **몬스터**(👾/👹), **링 배경**, **타격 이펙트**.

## 범위 경계 (이 묶음에서 지키는 선)

- **표시 전용**: `src/game/`·`formulas`·`constants`·스토어 액션 무변경. `BALANCE_VERSION`/`SCHEMA_VERSION` 불변(저장 키 `boxer-game.save.v8` 유지).
- **슬롯에 끼우기만**: TASK-026이 보존한 `data-animation-key`/`data-pose`/`data-boxer-type`/`data-gender` 구조를 그대로 사용. 새 게임 상태·애니 키를 만들지 않는다.
- **MVP 경계**: 신규 기능·강화 축·탭 추가 없음. 월드맵 동작은 여전히 `TODO`.

## 공통 규칙 (CLAUDE.md / AGENTS.md)

- **검증 게이트**: 변경 후 `node tools/check.mjs fast "<바뀐 파일>"`. UI/CSS 변경이므로 `npm run e2e`를 반드시 추가로 돌린다(testid·data-속성 보존이 최대 리스크).
- **버전 관리**: 표시 전용이라 범프 없음. 합성 중 저장 형태/수식을 건드리면 즉시 중단·재검토.

## 의존성 메모

- **선행 완료 전제**: TASK-026(단일 무대 합성)이 머지된 상태에서 착수한다.
- **에셋 의존**: 몬스터·링 배경·포즈 시트는 **에셋이 확정돼야** 착수 가능(현재 미보유). 명세는 TASK-027 §에셋 명세 참조.
