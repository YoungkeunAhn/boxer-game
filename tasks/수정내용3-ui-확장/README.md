# 구현 태스크 (수정내용3: 예시 UI 확장 — Phase P2~P5)

`docs/ui/`(예시 이미지 목업 기반 갭 분석 + 2026-06-21 범위 결정)를 구현 가능한 단위로 쪼갠 배치다.
`수정내용2`(TASK-004~013)가 완료 기준선이며, 이 폴더는 그 위에 **표현·UX(P2)**와 **진행/재화(P3)**를 얹고, **장비(P4)·상점결제/경기장(P5)**은 보류 골격으로만 둔다.

> 파트별 폴더는 `docs/ui/`의 파트 번호(01~07)와 1:1로 매핑된다. 각 task의 상세 기획 근거는 같은 번호의 `docs/ui/0N-*.md`에 있다.

## Phase 도입 전략 (docs/ui/README.md 확정)

한 번에 다 넣지 않는다. 저장 스키마는 **P3에서 1회만** 흔든다(v6→v7). MVP 경계는 단계적으로 해제한다.

| Phase | 포함 | 저장 영향 | 상태 |
| --- | --- | --- | --- |
| **P2 — 표현·UX** | 전투 헤더·적 카드·진행바, AUTO·수동스킬·배속, 강화 9종 그룹 UI, 타입 외형 전환, 6포즈 애니 | 없음~소 | **채택** |
| **P3 — 진행/재화** | 다이아·플레이어 레벨/경험치, 5탭 네비, 퀘스트 | 🔴 큼 (1회, v6→v7) | **채택** |
| **P4 — 성장 깊이** | 가방/장비·훈련 | 🔴 큼 | **보류(골격)** |
| **P5 — 수익화/경쟁** | 상점(무과금만, 결제 보류)·경기장(PVP) | 🔴 큼 | **보류(골격)** |

권장 순서 = **P2 → P3 → P4 → P5**. P5는 정책·심의·서버 검토 전 착수 금지.

## 모든 태스크 공통 규칙 (CLAUDE.md / AGENTS.md)

- **순수 함수 유지**: 전투·피해·보상·전이·강화·정산·시간(일일 리셋·경험치)은 `src/game/`의 순수 함수로 구현하고 `now`/`random`/`schedule`은 스토어에서 주입한다. `Date.now`/`Math.random`/타이머 직접 사용 금지.
- **버전 관리**: 저장 형태(`Boxer`/`SaveData`) 변경 → `SCHEMA_VERSION` 범프 + 저장 키 접미사(`boxer-game.save.vN`) 일치. 수식·밸런스 변경 → `BALANCE_VERSION` 범프. **P3 재화·레벨·퀘스트는 한 번의 v6→v7 범프로 합친다.** 옛 저장은 삭제하지 않고 `legacy`/`invalid` 안내.
- **미확정 수치**: 기획에 `가정:`/`TODO`로 남은 수치(경험치 곡선·다이아 보상·타입 전환 비용 등)는 `constants.ts`에 `가정:` 주석과 함께 임시값으로 넣고, 같은 값을 기획 문서/테스트와 동기화한다.
- **테스트**: 신규 순수 함수는 colocated `*.test.ts`, 스토어 변경은 `createGameStore` + 가짜 `now`/`random`/`schedule`로 검증.
- **검증 게이트**: `node tools/check.mjs fast "<바뀐 파일>"`. 설정·번들·의존성 변경이나 최종 검증은 `node tools/check.mjs full`. UI/CSS/스토어 변경 시 `npm run e2e` 추가.
- **MVP 경계**: 결제(IAP)·광고·길드·서버 동기화는 범위 밖. 장비·훈련(P4)·PVP/랭킹(P5)은 보류 — 골격 문서만 유지하고 구현하지 않는다.
- **mvp-scope 갱신**: 다이아·플레이어 레벨·퀘스트 채택(P3) 시 `docs/기획/overview/mvp-scope.md`의 "제외" 목록에서 해당 항목을 해제한다(에너지·결제·장비·경기장은 그대로 제외/보류).

## 의존성 메모

- **P2는 저장 스키마 무관** → 먼저 착수 가능. TASK-014~017(02 파이터)·TASK-018(07 애니)은 서로 독립적이나, TASK-015(AUTO/배속)는 전투 로직을 건드리므로 회귀 테스트·E2E 비중이 크다.
- **TASK-019(재화·레벨)가 P3의 토대** — TASK-020(5탭 헤더의 재화 표시)·TASK-021(퀘스트 보상)이 여기에 의존한다. P3는 TASK-019 → 020 → 021 순서 권장.
- **TASK-018(6포즈)**은 TASK-017(타입 전환)이 노출하는 타입 상태에 외형을 연결한다(전환 시 6포즈 세트 교체).
- **보류(TASK-022/023/024)**는 구현 의존이 아니라 문서 유지용. P4/P5 착수 결정 전에는 코드 변경 없음.
