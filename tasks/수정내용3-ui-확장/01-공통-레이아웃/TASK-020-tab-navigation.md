# TASK-020 5탭 네비게이션·상단 바 (P3)

## 목표

공통 프레임(상단 바 + 하단 5탭 네비)을 추가한다. 상단 바는 프로필·플레이어 레벨/경험치 바 + 재화(골드·다이아) + 일일 리셋 타이머를 표시하고, 하단은 5탭(상점·가방·파이터·퀘스트·경기장) 전환을 제공한다. **TASK-019(재화·레벨) 완료 후** 착수한다.

## 참고 문서

- `docs/ui/01-공통-레이아웃.md` §3-1, §3-2
- `docs/ui/03-상점.md`·`docs/ui/04-가방-장비.md`·`docs/ui/06-경기장-리그.md`(보류 탭 잠금), `docs/ui/05-퀘스트.md`(뱃지)
- `src/pages/HomePage.tsx`, `src/components/`

## 작업 범위

1. **상단 바**: 좌측 프로필 아바타·이름·플레이어 레벨(Lv)·경험치 바(%). 우측 재화 `[🪙골드][💎다이아][⏱일일리셋]`. 4재화→2재화+타이머로 360px 가로 압박 완화. 값은 TASK-019 셀렉터에서 파생.
2. **하단 5탭**: `[상점][가방][파이터][퀘스트][경기장]`, 중앙 **파이터가 홈/기본 탭**. 선택 탭 강조(빨강)+라벨.
3. **탭 전환**(`가정:` 라우터 미도입 — `HomePage`에 탭 상태만 추가). 파이터=메인 전투(기존 화면), 퀘스트=TASK-021 화면.
4. **보류 탭 잠금**: 가방(P4)·경기장(P5)·상점(P5 — 무과금 골격만)은 **숨김 또는 비활성(잠금 표시)**. 5탭 자리는 유지하되 진입 차단(`가정:`).
5. **알림 뱃지**: 수령 가능한 보상/신규가 있을 때만 빨강 점. 상점=무료 상자 수령 가능 시, 퀘스트=완료·미수령/마일스톤 수령 가능 시(TASK-021). 가방·경기장=보류 → 뱃지 없음.
6. **일일 리셋 타이머**(⏱): 다음 로컬 00:00까지 카운트다운, TASK-019 순수 파생(주입 now) 재사용.
7. E2E: 탭 전환·보류 탭 잠금·재화/레벨 헤더·뱃지 표시 갱신.

## 구현 원칙

- 프레젠테이셔널: 헤더/탭은 스토어 셀렉터로만 구동. 타이머는 주입 now 파생(`Date.now` 금지).
- 보류 탭은 라우팅 진입을 막되 자리를 유지(향후 해제 대비).

## 하지 않을 것

- 가방/경기장 화면 구현(보류). 상점 구매 로직(TASK-023 골격).
- 신규 저장 필드(탭 상태는 휘발 UI 상태). 라우터 라이브러리 도입(탭 상태만).

## 완료 기준

- 상단 바가 레벨·경험치·골드·다이아·일일 타이머를 정확히 표시한다.
- 5탭 전환이 동작하고 파이터/퀘스트만 진입 가능, 나머지는 잠금 표시된다.
- 알림 뱃지가 수령 가능 조건에서만 노출된다.
- 360px·데스크톱 한 열에서 깨지지 않는다.
- `node tools/check.mjs fast "<바뀐 파일>"` + `npm run e2e` 통과.
