# TASK-015 전투 컨트롤 — AUTO·수동 스킬·배속 (P2)

## 목표

전투 컨트롤 3종을 도입한다: **AUTO 토글**, **수동 스킬 버튼**, **배속(x2)**. 순수 로직(`resolveAttack`/`advanceCombat`)은 재사용하고, 입력·모드만 스토어 액션/상태로 분리한다. 밸런스 수치는 불변(보스 타임아웃은 게임 시간 기준)이라 `BALANCE_VERSION`은 올리지 않되, **전투 로직을 건드리므로 회귀 테스트·E2E가 핵심**이다.

- `docs/ui/02-파이터-메인화면.md` §3-2, §3-6
- `docs/기획/skills/equip.md`(자동 발동 규칙 — "AUTO 상태별 발동 규칙" 절 추가 필요), `docs/기획/combat/boss.md`(`bossTimeLimitMs`)
- `src/game/combat.ts`, `src/stores/gameStore.ts`

## 작업 범위

1. **AUTO 토글**:
   - AUTO ON = 현행 자동 전투 타이머 유지.
   - AUTO OFF = 자동 틱 정지 + **수동 탭 공격**(입력 액션마다 `resolveAttack` 1회). "유일 타이머" 구조는 유지하며 "OFF면 스케줄 정지 + 입력 액션으로 1틱"으로 확장.
2. **수동 스킬 버튼**(쿨다운 표시): AUTO 상태에 종속.
   - AUTO ON = 쿨 종료 시 자동 발동(현행 `equip.md` 유지).
   - AUTO OFF = 쿨 완료된 스킬을 버튼으로 수동 발동. 슬롯 우선순위(Slot1>2>3) 동일.
   - `docs/기획/skills/equip.md`에 "AUTO 상태별 발동 규칙" 절을 추가한다.
3. **배속 x2**: `advanceCombat`의 경과 시간에 배율 적용(게임 시간 가속). **보스 타임아웃은 게임 시간 기준**이라 배속과 무관하게 동일 게임 시간(실시간만 단축) → 밸런스 불변. 배율 단계는 `가정:` x1/x2.
4. 컨트롤 UI(AUTO 토글·스킬 버튼·배속 토글)를 하단/우측에 프레젠테이셔널 컴포넌트로 추가.
5. 테스트:
   - AUTO OFF에서 입력 액션 1회 = `resolveAttack` 1틱(자동 틱 없음).
   - 배속 x2에서 같은 게임 시간 경과 시 보스 타임아웃 결과가 x1과 동일(밸런스 불변) 회귀.
   - AUTO 상태별 스킬 발동(ON=자동, OFF=수동) 회귀.
6. E2E: AUTO 토글·배속·수동 스킬 동작을 RNG/clock 동결로 안정화해 추가.

## 구현 원칙

- `Date.now`/`Math.random`/타이머 직접 사용 금지 — 주입 유지.
- 순수 함수는 변이 없이 새 객체 반환. 입력·모드는 스토어 상태로만.
- 보스 타임아웃은 반드시 게임 시간 기준으로 둬서 배속이 밸런스를 바꾸지 않게 한다.

## 하지 않을 것

- 밸런스 수치 변경(`BALANCE_VERSION` 불변).
- x4 이상 배속(현재 x1/x2만, 확장은 추후).
- 저장 스키마 변경(AUTO/배속/모드 상태는 휘발 UI 상태로 충분한지 검토 — 영속 필요 시에만 P3 범프에 합류).

## 완료 기준

- AUTO ON/OFF 전환이 자동 타이머↔수동 탭으로 정확히 동작한다.
- 수동 스킬 버튼이 AUTO OFF에서만 활성·쿨다운대로 발동한다.
- 배속 x2가 실시간만 단축하고 보스 타임아웃/밸런스는 불변임을 테스트로 증명한다.
- `node tools/check.mjs fast "<바뀐 파일>"` + `npm run e2e` 통과.
