# TASK-016 강화 패널 9종 그룹/탭 정리 (P2)

## 목표

강화 패널을 **현행 9종 유지**한 채로, 360px 한 열에서 보기 좋게 **공격 계열 / 방어 계열 그룹(또는 탭)**으로 정돈한다. 이미지의 6칸 재구성(스킬강화·장비강화·훈련강화 신규 축)은 **미채택**이다. 수식·비용 곡선·상한·저장 형식은 **변경하지 않는다** → `BALANCE_VERSION`/`SCHEMA_VERSION` 불변(순수 표현 변경).

## 참고 문서

- `docs/ui/02-파이터-메인화면.md` §3-3, §3-5
- `docs/ui/04-가방-장비.md`(장비강화 축 미채택 → 02와 커플링 해소)
- `src/game/types.ts`(`UpgradeLevels`), `src/game/formulas.ts`(`upgradeCost`)

## 작업 범위

1. 9종을 두 그룹으로 묶어 노출(라벨 한글 유지):
   - **공격 계열**: 공격력`attackPower` · 공격속도`attackSpeed` · 치명타율`critRate` · 치명타피해`critDamage` · 골드보너스`goldBonus`
   - **방어 계열**: 체력`maxHp` · 방어`defense` · 회피`dodge` · 카운터`counter`
2. 각 강화 칸: 아이콘 · 이름 · 레벨 · 현재값 · **다음 레벨 증가량(+표시)** · 비용 버튼(잔액 부족/상한 비활성). 증가량·비용은 기존 셀렉터/`upgradeCost`에서 파생.
3. 그룹 전환 UI(탭 or 접기)로 360px 세로 길이를 줄인다.
4. E2E: 9종이 그룹/탭 안에서 모두 노출·구매되는지 갱신.

## 구현 원칙

- 표현만 변경. `upgradeCost(key, level) = ceil(UPGRADE_BASE_COSTS[key] * 1.25^level)` 그대로.
- 신규 강화 축(스킬·장비·훈련강화) 도입 금지(P4 장비/훈련 보류와 일관).

## 하지 않을 것

- 강화 종류 추가/삭제/병합, 비용·상한·수식 변경.
- `BALANCE_VERSION`/`SCHEMA_VERSION` 범프.

## 완료 기준

- 9종이 공격/방어 그룹으로 정돈되어 360px에서 가로 스크롤 없이 노출·구매된다.
- 증가량(+)·비용·비활성 상태가 기존 로직과 일치한다.
- `node tools/check.mjs fast "<바뀐 파일>"` + `npm run e2e` 통과.
