# TASK-024 경기장·PVP (P5 — 보류 골격)

> ⏸ **구현 보류(P5, 최후순위).** 2026-06-21 결정으로 **경기장(PVP)은 구현하지 않는다.** 서버가 없어 진짜 동기 대인전이 불가하다. 도입한다면 **"더미 상대 + 로컬 시뮬"** 형태만 실현 가능하다. 또한 에너지(⚡) 미채택이므로 도전 비용(⚡) 부분은 에너지 도입 결정과 함께 재설계해야 한다. 이 문서는 **착수 결정 시 출발점**으로만 유지한다. 현재 코드 변경 없음.

## 보류 사유

- `docs/기획/overview/mvp-scope.md`에서 **PVP·랭킹·리그·승률·상대 선택 모두 명시적 제외**.
- 서버 동기화 부재 → 진짜 대인 동기 매칭 불가. 트로피·시즌·순위는 더미 분포로만 표현 가능.
- PVP 전투 판정 = 신규 순수 로직 모듈 + `BALANCE_VERSION`, 트로피·시즌·전적 = 저장 스키마 추가 → `SCHEMA_VERSION`.

## 도입(P5) 착수 조건 — 가장 후순위

- P2~P4가 안정화되고, PVP 판정 방식(아래)·서버 유무 정책이 결정된 뒤.
- 에너지(⚡) 도입 여부 결정(도전 비용 재설계 전제).
- mvp-scope의 PVP·랭킹 제외 전면 재정의.

## 도입 시 채울 섹션 (참고: `docs/ui/06-경기장-리그.md` §3)

- [ ] **PVP 전투 판정**(핵심): 비동기(상대 스냅샷 vs 내 캐릭터 자동 시뮬) 권장. 방식 A=더미 상대 스냅샷 vs `calculateCombatStats`로 기존 전투 로직 간이 시뮬(주입 random)→승패→트로피. 방식 B=간이 승패 공식(power+난수). 결정성(주입 난수) 필수.
- [ ] 상대 데이터: 서버 없으니 **봇/더미 상대** 생성 규칙(내 트로피 ± 범위 능력치 스냅샷 합성).
- [ ] 리그/트로피/시즌: 트로피 증감(승 +N/패 -M), 티어·승급/강등, 시즌 길이·리셋·보상, 순위(더미 분포 추정 or 보류).
- [ ] 도전/방어: 도전 비용(에너지 도입 시), 상대 갱신(무료 N회+다이아), 방어 팀 의미(무서버면 기록 시뮬).
- [ ] 기록/랭킹 화면 데이터 범위.
- [ ] 데이터 모델 초안(`ArenaState`/`OpponentSnapshot`)은 `docs/ui/06-경기장-리그.md` §3-5 참조.
