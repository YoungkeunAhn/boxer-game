# TASK-018 타입별 6포즈 애니메이션 (P2)

## 목표

타입/성별 4종 각 **6포즈** 스프라이트를 게임 상태에 매핑한다. 4종은 한 캐릭터의 **교체 가능한 외형**이므로(TASK-017), 타입 전환 시 6포즈 세트 전체가 함께 바뀐다. 순수 표현 계층(에셋/CSS)으로 전투 로직·저장·밸런스에 영향 없다. **신규 애니 키 `boxer_counter` 1개만 추가**된다(카운터 전용 애니 미정의).

- `docs/기획/combat/hp-and-defeat.md`(GUARD), `docs/기획/combat/monster-attacks.md`(회피 판정)

## 작업 범위

1. **포즈↔상태 매핑**(기존 키 최대 재사용):
   | # | 인파이터 | 아웃파이터 | 재생 시점 | 기존 키 |
   | --- | --- | --- | --- | --- |
   | 1 | 기본 가드 | 롱 가드 | idle | `boxer_idle` |
   | 2 | 타이트 가드 | 라이트 풋워크 | 피격 방어/대기변형 | `boxer_guard` |
   | 3 | 숏 잽 | 스텝 백(회피) | 인파=잽/아웃=회피 | `boxer_left_jab` / `boxer_dodge` |
   | 4 | 바디 훅 | 롱 잽 | 훅/잽 | `boxer_left_hook` / `boxer_left_jab` |
   | 5 | 어퍼컷 | 스트레이트 크로스 | 어퍼/스트레이트 | `boxer_*_upper` / `boxer_right_straight` |
   | 6 | 카운터 훅 | 카운터 펀치 | 카운터 발동 | **`boxer_counter`(신규)** |
2. **신규 키 `boxer_counter`**: 카운터 발동 시 재생. 카운터는 메커니즘만 있고 명명 애니가 없으므로 키·트리거를 추가한다.
3. **타입 정체성 연출**: 같은 "잽"도 인파이터=숏(근접·짧은 리치), 아웃파이터=롱(거리·긴 리치) — 동일 키를 외형/이동 거리로 차별화. 톤: 인파이터 적/묵직·화면 흔들림, 아웃복서 청/잔상·MISS·COUNTER 강조(`ui-tone.md` 연동).
4. **회피 모션 중복 정리**: 기존 회피 6종과 아웃파이터 "스텝 백/라이트 풋워크"의 중복을 정리(신규 키 만들지 말고 재사용 매핑 명확화).
5. 스프라이트 시트 사양은 `가정:`/`TODO`(4종 × 포즈별 N프레임, 트리밍 PNG + 프레임 메타 json). 프레임 수·해상도·360px 메모리 상한은 아트 확정 전 `TODO`. CSS 애니 vs 스프라이트 시트는 성능 측정 후 결정.
6. E2E: 카운터 발동 시 `boxer_counter` 연출, 타입 전환 시 6포즈 세트 교체 표시 갱신.

## 구현 원칙

- 순수 표현. 전투 판정·저장·밸런스 불변(`BALANCE_VERSION`/`SCHEMA_VERSION` 무관).
- 신규 키는 `boxer_counter` 하나로 제한 — 나머지는 기존 키 재사용.
- 에셋 미확정 동안 플레이스홀더로 매핑 구조부터 완성(아트 교체 가능하게).

## 하지 않을 것

- 전투 규칙·판정 변경(애니는 상태 표시만).
- 6포즈 외 신규 애니 키 추가(`boxer_counter` 외).
- 4캐릭터 개별 보유(외형 전환만 — TASK-017).

## 완료 기준

- 6포즈가 idle/가드/공격/회피/카운터 상태에 정확히 매핑된다.
- 카운터 발동 시 `boxer_counter`가 재생된다.
- 타입 전환 시 외형/6포즈 세트가 함께 교체된다.
- 회피 6종과 아웃파이터 풋워크 중복이 정리된다.
- `node tools/check.mjs fast "<바뀐 파일>"` + `npm run e2e` 통과.
