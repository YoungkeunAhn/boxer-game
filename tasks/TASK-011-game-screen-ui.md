# TASK-011 게임 화면형 UI 재구성 (v1.4a)

## 목표

상태 패널 위주 UI를 **게임 화면형 레이아웃**으로 재구성한다(상단 정보 / 중앙 전투 / 하단 조작). 앞 태스크가 노출한 상태(복서·몬스터 HP, 그로기, 스킬 쿨타임, 콤보, 방어 결과, 신규 강화)를 360px WebView 한 열로 표시한다.

## 참고 문서

- `docs/presentation/ui.md`, `docs/combat/hp-and-defeat.md`, `docs/combat/boss.md`
- `docs/browser-smoke-checklist.md`

## 작업 범위

1. 상단 패널: 현재 스테이지/챕터 이름/복서 타입/골드/보스 여부.
2. 중앙 전투 영역:
   - 왼쪽 복서 / 오른쪽 몬스터 배치(플레이스홀더 비주얼 허용).
   - 복서 HP 바 / 몬스터 HP 바 / (보스) 그로기 바.
   - 데미지 숫자, 스킬명 텍스트, MISS / GUARD / COUNTER 텍스트 표시 영역.
3. 하단 패널:
   - 기본 스킬 쿨타임 표시: 잽 / 스트레이트 / 훅 / 어퍼(다음 준비 시각 기반 진행도).
   - 전용 스킬 슬롯: Slot 1~3 + 패시브 표시(장착/해제는 TASK-010 액션 사용).
   - 강화 버튼: 기존 5종(공격력·공격속도·치명타율·치명타 피해·골드 보너스) + 신규 4종(체력·방어·회피·카운터). 비용·상한·잔액 부족 비활성 표시.
4. CSS Modules(`*.module.css`) + `src/styles/global.css`로 360px 한 열 레이아웃, 터치 영역 충분, 가로 스크롤 없음, safe-area 대응.
5. 컴포넌트 분리: 전투 영역 / 스킬·쿨타임 / 강화 패널을 프레젠테이셔널 컴포넌트로 분리하고 스토어 셀렉터로 구동.
6. E2E: `e2e/*.spec.ts`와 `docs/browser-smoke-checklist.md`를 신규 요소(HP 바·그로기 바·스킬 쿨타임·신규 강화)에 맞게 갱신.

## 구현 원칙

- UI는 프레젠테이셔널: 로직을 새로 만들지 않고 스토어 상태만 표시·디스패치한다.
- 정보 우선순위: HP/보스 타이머/그로기처럼 생존·진행에 직결되는 정보가 한눈에 보여야 한다.
- 360px에서 핵심 정보·버튼이 한 열로 깨지지 않아야 한다.

## 하지 않을 것

- 타입별 톤·애니메이션·이펙트(TASK-012).
- 전투 규칙 변경(앞 태스크에서 확정된 동작만 노출).

## 완료 기준

- 상/중/하 레이아웃에 HP·그로기·스킬 쿨타임·콤보·방어 결과·신규 강화가 모두 표시된다.
- 360px와 데스크톱에서 한 열로 깨지지 않고 가로 스크롤이 없다.
- `node tools/check.mjs full` + `npm run e2e`(데스크톱·모바일 360) 통과.
