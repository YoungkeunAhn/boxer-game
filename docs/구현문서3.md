# 작업 요청: TASK-003 앱인토스 WebView SDK 연동 및 샌드박스 실행 준비

현재 복서키우기 v0.1은 브라우저에서 핵심 게임 루프 구현을 목표로 진행 중이다.

이번 작업의 목표는 기존 Vite + React + TypeScript 프로젝트를 앱인토스 WebView 미니앱 환경에서 실행할 수 있도록 SDK와 설정을 연동하는 것이다.

이번 작업에서는 새로운 게임 시스템을 추가하지 않는다.

## 이번 작업 목표

앱인토스 WebView SDK를 연동하고, 기존 복서키우기 v0.1 게임이 일반 브라우저와 앱인토스 샌드박스 환경 모두에서 실행될 수 있는 상태를 만든다.

## 작업 범위

1. 앱인토스 WebView SDK 설치
2. 앱인토스 프로젝트 초기화
3. `granite.config.ts` 생성 또는 수정
4. 앱인토스 앱 설정값 정리
5. Vite dev/build 명령 연결
6. 게임 WebView 타입 설정
7. 기존 게임 루프 동작 유지 확인
8. localStorage 저장/불러오기 유지 확인
9. 모바일 WebView 실행을 고려한 기본 설정 점검
10. 샌드박스 앱 테스트 준비

## 설치 및 설정 참고

공식 앱인토스 문서를 확인해서 최신 설치 방법을 기준으로 진행한다.

예상 작업:

* `@apps-in-toss/web-framework` 설치
* `ait init` 실행
* `granite.config.ts` 설정
* `appName`, `displayName`, `primaryColor`, `icon` 값 정리
* `web.host`, `web.port`, `web.commands.dev`, `web.commands.build` 설정
* `outdir`를 Vite 빌드 결과물과 맞춤
* 게임 앱이므로 WebView 타입을 game으로 설정

## 구현 원칙

* 기존 게임 로직을 변경하지 않는다.
* `src/game` 내부의 순수 함수 구조를 유지한다.
* Zustand store 구조를 유지한다.
* localStorage 저장 키가 변경되지 않게 주의한다.
* 앱인토스 설정은 게임 로직과 분리한다.
* SDK 연동 때문에 브라우저 실행이 깨지지 않게 한다.
* 앱인토스 환경에서만 필요한 코드는 별도 모듈로 분리한다.
* 공식 문서 기준이 불명확한 부분은 임의 구현하지 말고 TODO로 남긴다.

## 점검할 기능

1. 일반 브라우저 실행

   * `npm run dev` 또는 기존 dev 명령으로 실행되는가
   * 복서 상태 화면이 정상 표시되는가
   * 훈련 버튼이 동작하는가
   * 경기 버튼이 동작하는가
   * 저장/불러오기가 유지되는가

2. 앱인토스 설정

   * `granite.config.ts`가 정상 생성되었는가
   * appName/displayName이 설정되었는가
   * Vite dev/build 명령이 연결되었는가
   * outdir이 Vite 빌드 결과물과 일치하는가
   * WebView 타입이 게임용으로 설정되었는가

3. 빌드 확인

   * TypeScript 에러 없이 빌드되는가
   * Vite 빌드 결과물이 생성되는가
   * 앱인토스 번들 생성 단계로 넘어갈 수 있는가

4. 모바일 WebView 기본 점검

   * 360px 너비에서 화면이 깨지지 않는가
   * 버튼 터치 영역이 충분한가
   * 가로 스크롤이 생기지 않는가
   * safe-area 대응이 필요한 부분이 있는가

## 완료 기준

* 앱인토스 설정 파일이 생성 또는 수정된다.
