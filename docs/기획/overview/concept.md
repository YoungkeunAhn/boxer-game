# 프로젝트 개요

## 게임 컨셉

플레이어는 자동으로 싸우는 복서를 골드와 성장 재화로 강화한다. 복서는 몬스터를 차례로 처치하고 각 장의 보스를 제한 시간 안에 쓰러뜨려 무한히 이어지는 다음 장으로 진출한다.

가정: 본 문서는 `수정내용2` 설계 방향(복서 타입·성별·스킬·복서 HP·몬스터 공격)을 반영한다. 현재 구현 기준선은 5종 능력치 자동 전투이며, 신규 시스템은 [복서 타입](../boxer/types.md)·[몬스터 공격](../combat/monster-attacks.md)·[체력과 실패](../combat/hp-and-defeat.md)에서 다룬다.

## 장르와 플랫폼

- 장르: 방치형 자동 전투 / 복싱 성장 게임
- 플레이 단위: 전투는 자동 진행하고, 플레이어는 짧게 접속해 강화와 보스 재도전을 결정한다.
- 플랫폼: React + Vite + TypeScript 기반 앱인토스 WebView 미니앱
- 저장: MVP는 싱글 플레이와 기기 `localStorage`를 사용한다.

## 타겟 경험

- 공격 주기마다 몬스터 HP가 줄어드는 즉각적인 성장 가시성
- 처치 골드를 어느 능력치에 투자할지 고르는 가벼운 전략성
- 보스 제한 시간으로 생기는 명확한 성장 목표
- 접속하지 않은 동안에도 현재 일반 스테이지를 반복해 얻는 방치 보상
- 3개 테마가 순환하며 수치가 커지는 끝없는 진행감

## 반복 행동

1. 복서가 현재 몬스터를 공격하고 골드를 획득한다.
2. 일반 몬스터를 처치하면 다음 스테이지로 자동 이동한다.
3. 골드로 공격력·공격속도·치명타율·치명타 피해·골드 보너스를 강화한다.
4. 각 장의 5번째 스테이지에서 30초 보스에 도전한다.
5. 실패하면 같은 장의 4번째 스테이지를 반복 파밍하고 준비가 되면 재도전한다.
6. 보스를 처치하면 다음 장으로 이동하고, 3개 테마를 순환하며 반복한다.

## 설계 원칙

- 사용자의 연속 클릭 없이 전투가 계속 진행되어야 한다.
- 한 화면에서 몬스터 HP, 현재 장-스테이지, 보스 시간, 골드와 강화 선택을 파악할 수 있어야 한다.
- 보스 실패는 진행 손실이 아니라 강화에 필요한 반복 파밍 상태로 전환한다.
- 복서 HP가 0이 되어도 게임오버가 아니라 현재 스테이지에 머문다. → [체력과 실패](../combat/hp-and-defeat.md)
- 몬스터 공격은 복서 타입(회피/가드) 차이를 드러내는 장치로 사용한다. → [몬스터 공격](../combat/monster-attacks.md)
- 확률은 치명타·회피·카운터 판정에 사용하고 골드 보상과 강화 비용은 예측 가능하게 표시한다.
