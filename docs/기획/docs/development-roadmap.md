# 개발 로드맵

## 원칙

- 각 버전은 실행 가능한 게임 상태로 끝낸다.
- 산식과 콘텐츠는 코드에서 분리하고 핵심 로직에 자동 테스트를 둔다.
- 일정과 담당자는 `TODO: 팀 구성 및 기술 스택 확정 후 산정`한다.

## v0.1 — 핵심 루프

목표: 훈련부터 경기 보상까지 최소 루프를 플레이할 수 있다.

1. 데이터 구조와 콘텐츠 로더 설계
2. 복서 생성 및 상태 화면 구현
3. 수동 훈련과 능력치 증가 구현
4. 상대 데이터와 경기 결과 계산 구현
5. 돈·명성 보상과 기본 해금 구현
6. 핵심 산식 단위 테스트 작성

완료 조건: 복서를 생성해 첫 상대에게 승리하고 보상을 확인할 수 있다.

## v0.2 — 성장 선택

목표: 보상을 소비해 성장 전략을 선택할 수 있다.

1. 장비 목록, 구매, 장착 구현
2. 장비 효과와 전투력 재계산 구현
3. 상대 및 훈련 콘텐츠 확장
4. 코치 데이터 준비
5. 밸런스 시뮬레이션과 초기 수치 조정

완료 조건: 서로 다른 장비 선택이 훈련 효율 또는 승률에 반영된다.

## v0.3 — 방치와 유지

목표: 자동 성장과 안정적인 진행 복구를 제공한다.

1. 자동 훈련 해금 및 정산 구현
2. 오프라인 보상과 최대 인정 시간 구현
3. 저장/불러오기와 원자적 저장 구현
4. 저장 스키마 마이그레이션 구현
5. 첫 플레이 안내와 오류 흐름 보완

완료 조건: 재접속 시 저장 상태와 오프라인 보상이 중복 없이 복원된다.

## v1.0 — MVP 출시 후보

목표: 전체 성장 루프를 안정적으로 완주할 수 있다.

1. 코치 시스템 최종 구현
2. 리그 progression과 세계 챔피언 목표 구현
3. UX, 접근성, 사운드, 연출 개선
4. 밸런스·성능·저장 안정성 테스트
5. 분석 이벤트와 오류 로깅 추가
6. 배포 환경 및 운영 체크리스트 확정

완료 조건: 신규 플레이부터 자동 훈련 해금과 최종 목표까지 치명적 오류 없이 진행된다.

## MVP 이후 후보

- 추가 능력치와 전투 시뮬레이션 확장
- 스토리, PVP, 길드, 랭킹
- 시즌, 광고, 결제는 별도 사업·운영 기획 후 검토

## 주요 위험

- 콘텐츠 부족: 데이터 추가 비용이 낮은 구조와 밸런스 도구를 먼저 만든다.
- 성장 정체: 목표 시간과 이탈 구간을 측정해 가격·보상을 조정한다.
- 시각 조작: 오프라인 경과 시간 상한과 비정상 시각 검사를 적용한다.
- 저장 손상: 원자적 저장, 백업, 버전별 마이그레이션을 제공한다.
