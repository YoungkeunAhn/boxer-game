# 능력치와 수식

## 기본 능력치

| 능력치 | 키 | 역할 |
|---|---|---|
| 체력 | `health` | 버티는 힘을 나타내며 전투력에 30% 반영 |
| 공격력 | `attack` | 피해 능력을 나타내며 전투력에 40% 반영 |
| 방어력 | `defense` | 피해 감소 능력을 나타내며 전투력에 20% 반영 |
| 스피드 | `speed` | 반응과 유효타 기회를 나타내며 전투력에 10% 반영 |

## 전투력

```text
전투력 = 체력 × 0.3 + 공격력 × 0.4 + 방어력 × 0.2 + 스피드 × 0.1
```

- 장비 효과를 적용한 유효 능력치를 사용한다.
- 화면 표시는 반올림하되 내부 계산은 소수 정밀도를 유지한다.
- 가정: 네 능력치의 성장 비용이 비슷하므로 가중치 합을 1로 둔다.

## 훈련 보상

```text
최종 증가량 = 기본 증가량 × 장비 배율 × 코치 배율 × 자동화 배율
```

- 각 배율은 기본값 `1.0`이며 같은 범주의 보너스는 먼저 합산한다.
- 예: 장비 보너스가 10%와 20%면 장비 배율은 `1 + 0.1 + 0.2 = 1.3`이다.

## 경기 승률

```text
기본 승률 = 내 전투력 / (내 전투력 + 상대 전투력)
최종 승률 = clamp(기본 승률, 최소 승률, 최대 승률)
승리 = random(0, 1) < 최종 승률
```

- 가정: 최소 승률은 5%, 최대 승률은 95%로 완전 확정 결과를 방지한다.
- 같은 전투력이면 승률은 50%다.

## 레벨업

```text
총 성장치 = (현재 네 기본 능력치 합) - (초기 네 기본 능력치 합)
필요 성장치(level) = 10 × level²
```

## 밸런싱 주의사항

- 공격력 가중치가 높아 단일 능력치 집중이 항상 최적이 되지 않는지 검증한다.
- 훈련 시간 대비 성장량과 장비 가격 대비 효과를 공통 효율 지표로 비교한다.
- 승률 50% 구간의 반복 패배가 이탈로 이어지지 않도록 목표와 피드백을 제공한다.
- 자동·오프라인 보상이 수동 플레이를 무의미하게 만들지 않도록 상한을 둔다.
- 지수형 비용이 콘텐츠 소모 속도를 지나치게 늦추지 않는지 시뮬레이션한다.
- 밸런스 상수는 코드가 아니라 데이터로 관리하고 버전을 기록한다.
