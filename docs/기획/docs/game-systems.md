# 게임 시스템

## 복서 능력치 시스템

- 기본 능력치는 체력, 공격력, 방어력, 스피드다.
- 기본 능력치와 장비 효과를 합산한 유효 능력치로 전투력을 계산한다.
- 스태미나, 멘탈, 집중력, 회복력은 MVP 이후 후보로 둔다.
- 가정: MVP에서는 복서 레벨과 능력치 성장을 분리하고 레벨은 진행 지표로만 사용한다.

## 훈련 시스템

- 훈련은 하나 이상의 능력치를 증가시키며 기본 보상과 실행 비용을 가진다.
- 샌드백·미트는 공격, 줄넘기·로드워크는 스피드/체력, 웨이트는 공격/체력, 가드는 방어, 스파링은 복합 성장에 초점을 둔다.
- 실행 전 비용과 예상 증가량, 실행 후 실제 증가량을 표시한다.

## 경기 시스템

- 도전 가능 조건을 만족한 상대만 선택할 수 있다.
- 복서와 상대 전투력을 승률 함수에 넣고 난수로 승패를 판정한다.
- 경기 결과에는 승패, 계산 승률, 보상, 해금 내역을 포함한다.
- 가정: MVP는 라운드별 시뮬레이션 없이 단일 확률 판정을 사용한다.

## 보상 시스템

- 최초 승리는 추가 보상 또는 다음 상대 해금을 제공한다.
- 보상 지급은 경기 결과 ID별 한 번만 처리해 중복 수령을 막는다.

## 장비 시스템

- 부위는 글러브, 복싱화, 트렁크, 마우스피스, 붕대, 샌드백으로 구분한다.
- 구매한 장비는 영구 소유하며 장착 시 고정 능력치 또는 배율 효과를 준다.
- 가정: MVP는 부위당 한 개만 장착하고 강화·분해·랜덤 옵션은 제외한다.

- 코치는 돈 또는 명성 조건으로 고용하고 한 명만 활성화한다.

## 오프라인 보상 시스템

- `min(현재 시각 - 마지막 정산 시각, 최대 인정 시간)`만큼 자동 훈련을 계산한다.
- 시스템 시각이 과거로 이동한 경우 경과 시간을 0으로 처리한다.
- 정산 내역에 경과 시간, 적용 훈련, 능력치 증가량을 표시한다.
- 가정: 오프라인 효율은 온라인 자동 훈련의 50%이며 최대 시간은 추후 확정한다.

## 공통 구현 원칙

- 콘텐츠 정의와 플레이어 진행 상태를 분리한다.
- 모든 재화·보상 변경은 단일 트랜잭션 함수로 처리한다.
- 산식 버전을 저장해 밸런스 변경 및 마이그레이션에 대비한다.
