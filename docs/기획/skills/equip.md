# 스킬 장착 구조

## 핵심

공통 기본 스킬은 항상 사용한다. 타입 전용 스킬은 장착 슬롯으로 제한한다.

```text
공통 기본 스킬 4개 = 항상 자동 사용
타입 전용 스킬   = 최대 3개 장착
패시브 스킬      = 패시브 슬롯 적용
```

가정: 스킬 장착은 `수정내용2` 설계 방향이며 슬롯 수·해금 조건은 미확정이다.

## 슬롯 구성

| 슬롯 | 내용 |
| --- | --- |
| 기본 스킬 | 잽 / 스트레이트 / 훅 / 어퍼 (고정, 항상 사용) |
| 전용 슬롯 1~3 | 타입 전용 액티브 스킬 |
| 패시브 슬롯 | 타입 전용 패시브 스킬 |

기본 스킬은 [공통 기본 공격](../combat/basic-attacks.md)을 참고한다.

## 인파이터 장착 예시

```text
기본:   잽 / 스트레이트 / 훅 / 어퍼
전용:   Slot 1 리버샷 / Slot 2 압박 / Slot 3 뎀프시롤
패시브: 철벽가드
```

→ [인파이터 전용 스킬](./infighter-skills.md)

## AUTO 상태별 발동 규칙 (TASK-015)

전투 컨트롤의 AUTO 토글 상태에 따라 스킬 발동 방식이 달라진다. 슬롯 우선순위(Slot1 > 2 > 3)는 두 상태에서 동일하다.

| AUTO 상태 | 발동 방식 |
| --- | --- |
| AUTO ON | 쿨 종료 시 **자동 발동**(현행 유지). 여러 슬롯이 동시에 쿨 완료면 Slot1 > 2 > 3 순으로 발동 |
| AUTO OFF | 쿨 완료된 스킬을 **버튼으로 수동 발동**. 우선순위는 동일(버튼이 Slot1부터 노출) |

가정/TODO(2026-06-21): **스킬 슬롯 시스템(TASK-010)이 아직 코드에 없다.** `types`/`gameStore`/`components` 어디에도 스킬 슬롯·쿨다운 상태가 없어 위 "Slot1>2>3 우선순위"를 구현할 대상이 없다. 따라서 TASK-015의 수동 스킬은 **콤보 게이지 소비형 임시 스킬 1종('피니시')** 으로 한정해 구현했다.

- 발동 조건: **AUTO OFF + 콤보 게이지 가득**(`COMBO_GAUGE_MAX`)일 때만 버튼 활성.
- 효과: 어퍼 강타(데미지 ×`FINISHER_DAMAGE_MULT`) 1회 후 게이지 0으로 소비.
- 위 슬롯 기반 규칙은 **TASK-010 스킬 슬롯 도입 시** 임시 피니시를 슬롯 스킬로 교체하며 정식 적용한다.

밸런스 영향: 컨트롤(AUTO/배속/수동)은 휘발 UI 상태이고 보스 타임아웃은 게임 시간 기준이라 `SCHEMA_VERSION`/`BALANCE_VERSION` 모두 불변.
