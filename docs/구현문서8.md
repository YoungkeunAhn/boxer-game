# 작업 요청: TASK-008 앱인토스 콘솔 입력값 확정 및 검토 요청 준비

현재 복서키우기 v0.1은 앱인토스 WebView 미니앱 출시를 목표로 개발 중이다.

이번 작업의 목표는 앱인토스 콘솔에 입력할 앱 정보, 출시 리소스, 게임 등급분류 자료, 빌드/번들 상태를 최종 정리해서 검토 요청 직전 상태로 만드는 것이다.

이번 작업에서는 새로운 게임 기능을 추가하지 않는다.

## 이번 작업 목표

앱인토스 콘솔에 입력할 정보와 제출 자료를 최종 확정한다.

## 작업 범위

1. 앱 기본 정보 확정
   - 앱 이름
   - appName
   - displayName
   - 한 줄 소개
   - 짧은 앱 설명
   - 긴 앱 설명
   - 카테고리
   - 고객 문의 이메일
   - 개발자/운영자 정보

2. 앱 리소스 확인
   - 앱 아이콘
   - 대표 썸네일
   - 스크린샷
   - 게임 로고
   - 기본 캐릭터 이미지
   - 리소스 사이즈 공식 문서 확인 필요 항목 정리

3. 게임 등급분류 자료 확인
   - 등급분류 증빙 자료 준비 여부 확인
   - 게임 설명 자료 준비 여부 확인
   - 주요 화면 스크린샷 준비 여부 확인
   - 플레이 영상 준비 여부 확인
   - 기능 목록 정리 여부 확인
   - 폭력성 표현 수위 설명 여부 확인
   - 결제/광고/확률형 아이템 없음 명시 여부 확인

4. 앱 설명 최종 검토
   - 실제 구현된 기능만 설명에 포함되어 있는지 확인
   - 인앱 결제, 광고, 리더보드, 친구 초대 문구가 잘못 들어가지 않았는지 확인
   - v0.1 제외 기능이 소개 문구에 포함되지 않았는지 확인
   - 과장된 표현을 제거하고 실제 플레이 가능한 기능 중심으로 정리

5. 빌드/번들 준비 확인
   - TypeScript 빌드 성공 여부 확인
   - Vite 빌드 성공 여부 확인
   - 앱인토스 번들 생성 가능 여부 확인
   - 빌드 산출물 경로 확인
   - granite.config.ts 설정 확인
   - appName/displayName 설정 확인

6. 검토 요청 전 체크리스트 정리
   - 브라우저 실행 확인
   - 토스 앱 테스트 환경 실행 확인
   - 훈련 → 경기 → 보상 → 저장 루프 확인
   - 재접속 시 데이터 유지 확인
   - 모바일 화면 깨짐 없음 확인
   - Safe Area 문제 없음 확인
   - 고객 문의 경로 존재 확인
   - 등급분류 관련 TODO 해결 여부 확인

## 이번 작업에서 하지 않을 것

- 신규 게임 기능 추가
- 신규 상대 추가
- 장비 시스템 확장
- 코치 시스템 확장
- 토스 로그인 연동
- 서버 저장 연동
- 인앱 결제 추가
- 광고 추가
- 리더보드 추가
- 친구 초대 추가
- 출시 검토 요청 버튼 클릭

## 수정 원칙

- 출시 직전 단계이므로 기능 추가보다 안정화에 집중한다.
- 실제 구현되지 않은 기능은 앱 설명에서 제거한다.
- 제출 자료에 누락된 항목이 있으면 출시 차단 이슈로 분리한다.
- 출시 후 개선 가능한 항목은 별도 TODO로 분리한다.
- 정책 관련 불확실한 내용은 공식 문서 확인 필요로 표시한다.

## 완료 기준

- 앱인토스 콘솔 입력값이 정리된다.
- 앱 설명 문구가 최종 검토된다.
- 앱 아이콘, 썸네일, 스크린샷 준비 상태가 확인된다.
- 게임 등급분류 자료 준비 상태가 확인된다.
- 빌드와 번들 생성 가능 여부가 확인된다.
- 검토 요청 전 남은 차단 이슈가 명확해진다.
