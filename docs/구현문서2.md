# 작업 요청: TASK-002 v0.1 브라우저 QA 및 모바일 화면 대응

현재 복서키우기 v0.1의 핵심 게임 루프 구현이 진행되어 있다.

## 이번 작업 범위

이번 작업에서는 새로운 게임 시스템을 추가하지 않는다.

하지 않을 것:

* 앱인토스 SDK 연동
* 토스 로그인 연동
* 서버 저장
* 결제
* 광고
* 리더보드
* 신규 장비 시스템
* 신규 코치 시스템
* 복잡한 애니메이션

## 구현/점검할 것

1. v0.1 핵심 루프 동작 확인

   * 복서 생성
   * 복서 상태 표시
   * 훈련 실행
   * 능력치 증가
   * 전투력 계산
   * 경기 실행
   * 승리/패배 결과 표시
   * 보상 지급
   * localStorage 저장
   * localStorage 불러오기

2. 예외 상황 점검

   * 저장 데이터가 없을 때 새 게임이 정상 생성되는지 확인
   * localStorage 데이터가 깨졌을 때 안전하게 초기화되는지 확인
   * 경기 결과가 undefined 또는 NaN으로 표시되지 않게 처리
   * 훈련/경기 버튼을 빠르게 연속 클릭해도 데이터가 비정상적으로 변하지 않게 처리

3. 모바일 화면 대응

   * 360px 너비에서도 핵심 UI가 깨지지 않게 정리
   * 버튼 터치 영역을 충분히 크게 유지
   * 복서 상태, 훈련, 경기 영역이 세로 화면에서 자연스럽게 배치되도록 수정
   * 가로 스크롤이 생기지 않게 처리

4. UI 최소 정리

   * 현재 돈, 명성, 레벨, 주요 능력치가 한눈에 보이게 정리
   * 훈련 결과와 경기 결과 메시지를 명확하게 표시
   * 새 게임/초기화 버튼이 있다면 실수 방지 확인 절차 추가

5. 개발 품질 점검

   * 게임 로직은 UI와 분리된 상태를 유지
   * 전투력 계산, 훈련 처리, 경기 결과 계산은 순수 함수 구조 유지
   * UI 컴포넌트는 store의 액션을 호출하는 방식 유지
   * 저장/불러오기 로직은 별도 모듈에서 관리

## 완료 기준

* 브라우저에서 v0.1 게임 루프가 끊기지 않고 동작한다.
* 훈련 → 능력치 증가 → 경기 → 보상 → 저장 흐름이 정상 동작한다.
* 새로고침 후에도 복서 데이터가 유지된다.
* 모바일 세로 화면에서 플레이 가능한 UI가 된다.
* 앱인토스 SDK 연동 전 단계로 넘어갈 수 있는 상태가 된다.
