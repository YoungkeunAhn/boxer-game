# 출시 체크리스트

## 문서 목적

복서키우기 v0.1의 개발 완료부터 앱인토스 제출 직전까지 필요한 작업을 한곳에서 점검한다. 정책과 SDK 요구사항은 제출 시점에 다시 확인한다.

## 출시 단계 요약

- [ ] MVP 기능과 제외 범위를 확정한다.
- [ ] 개발 완료 기준을 충족하고 실제 기기 QA를 진행한다.
- [ ] 앱인토스 연동과 제출 리소스를 준비한다.
- [ ] 등급분류 및 검수 요건을 확인한 뒤 제출본을 고정한다.

## 개발 완료

- [ ] 복서 생성, 상태 조회, 훈련, 경기, 보상 흐름이 연결되어 있다.
- [ ] 전투력, 훈련, 경기 결과 계산이 UI와 분리되어 있다.
- [ ] 잘못된 입력이나 데이터에도 화면이 중단되지 않는다.
- [ ] 로딩, 빈 상태, 오류 상태가 사용자에게 표시된다.
- [ ] 프로덕션 빌드에서 오류와 불필요한 디버그 로그가 없다.
- [ ] v0.1 기능과 실제 구현 범위가 일치한다.

## 앱인토스 연동

- [ ] 게임이 토스 앱 안에서 정상 실행된다.
- [ ] 앱인토스 SDK 버전과 필수 설정을 확인했다. (`공식 문서 확인 필요`)
- [ ] SDK 초기화 실패 시 안내 또는 안전한 대체 동작이 있다.
- [ ] 뒤로가기, 백그라운드 전환, 복귀, 종료 흐름을 확인했다.
- [ ] 실제 기기와 토스 앱 QR 테스트를 완료했다. (`공식 문서 확인 필요`)
- [ ] 허용 도메인, 네트워크, 번들 규칙을 확인했다. (`공식 문서 확인 필요`)

## 저장 및 불러오기

- [ ] 새로고침 또는 재접속 후 데이터가 유지된다.
- [ ] 종료, 이탈, 백그라운드 전환 상황에서 데이터 손실이 없다.
- [ ] 최초 실행, 기존 저장 데이터, 손상된 데이터 흐름을 각각 테스트했다.
- [ ] 저장 스키마 버전과 마이그레이션 또는 초기화 정책이 정의되어 있다.
- [ ] 저장 실패 시 게임이 멈추지 않고 사용자에게 안내한다.
- [ ] localStorage 용량 초과와 사용 불가 상황을 처리한다.

## QA

- [ ] 게임 진행이 막히는 구간이 없다.
- [ ] 훈련 비용, 능력치 상승, 경기 보상 계산이 기획값과 일치한다.
- [ ] 연속 탭이나 빠른 화면 전환으로 보상이 중복 지급되지 않는다.
- [ ] 작은 화면, 긴 문구, 화면 회전 정책을 확인했다.
- [ ] 지원 대상 OS와 기기 범위에서 핵심 루프를 반복 테스트했다. (`공식 문서 확인 필요`)
- [ ] 네트워크가 느리거나 끊겨도 저장 데이터가 손상되지 않는다.
- [ ] 사운드가 있다면 On/Off 설정과 재접속 후 설정 유지를 확인했다.
- [ ] 치명적 오류, 재현 절차, 수정 여부를 릴리스 기록에 남겼다.

## 리소스 준비

- [ ] 앱 아이콘, 썸네일, 스크린샷, 설명 문구가 준비되었다.
- [ ] 리소스 크기와 파일 형식이 제출 기준에 맞는다. (`공식 문서 확인 필요`)
- [ ] 저작권 또는 사용 권한을 확인한 이미지, 폰트, 사운드만 포함했다.
- [ ] 고객 문의 이메일 또는 문의 경로가 준비되었다.
- [ ] 개인정보 처리 관련 안내 필요 여부를 확인했다. (`공식 문서 확인 필요`)

## 검수 제출 전

- [ ] 공식 앱인토스 검수 기준 확인을 완료했다. (`공식 문서 확인 필요`)
- [ ] 게임 등급분류 경로와 제출 가능 조건을 확인했다. (`공식 문서 확인 필요`)
- [ ] 검수 계정 또는 별도 진입 방법이 필요하면 준비했다.
- [ ] 제출 버전, 빌드 번호, 변경 사항을 기록했다.
- [ ] 제출본과 QA 완료본의 커밋 및 빌드가 동일하다.

## v0.1 제외 항목

- [ ] 결제, 광고, 리더보드, 친구 초대 코드가 노출되지 않는다.
- [ ] PVP, 길드, 시즌패스, 복잡한 스킬 시스템 진입점이 없다.
- [ ] 랜덤박스와 유료 확률형 아이템이 포함되지 않는다.
- [ ] 서버 계정 저장은 `TODO`로 유지하고 localStorage만 사용한다.

- [ ] 출시 차단 버그가 0건이고 잔여 이슈의 영향과 대응을 기록했다.
- [ ] 운영 담당자, 문의 대응 경로, 롤백 기준을 정했다. (`TODO`)
