# 앱인토스 출시 전략

## 문서 목적

복서키우기 v0.1을 앱인토스 미니앱으로 출시하기 위한 범위와 기본 방향을 정리한다.

## 출시 플랫폼

- 복서키우기는 토스 앱인토스 미니앱 출시를 목표로 한다.
- 앱인토스 지원 환경, 제출 절차, 검수 기준은 출시 시점의 `공식 문서 확인 필요`.

## 개발 방식

- WebView 기반 미니앱으로 개발한다.
- React + Vite + TypeScript를 사용한다.
- SSR은 사용하지 않고 CSR 또는 정적 빌드 방식으로 구성한다.
- 앱인토스 SDK는 게임 코드와 분리된 어댑터를 통해 연동한다.

## v0.1 출시 범위

- 복서 생성
- 훈련
- 경기
- 보상
- 저장/불러오기
- 재접속 시 진행도 유지

## v0.1 제외 범위

- 인앱 결제
- 인앱 광고
- 리더보드
- 친구 초대
- 복잡한 사운드
- 서버 기반 계정 저장

## 출시 전 필수 확인

- 앱인토스 SDK 연동 요구사항 확인 (`공식 문서 확인 필요`)
- 앱 번들 생성 및 실제 기기 실행 확인
- 토스 앱 QR 테스트 절차 확인 (`공식 문서 확인 필요`)
- 게임 등급분류 준비
- 앱 아이콘과 썸네일 준비
- 고객 문의 이메일 또는 문의 경로 준비
- 앱인토스 검수 체크리스트 확인 (`공식 문서 확인 필요`)
