# MVP 범위

## v0.1 목표

신규 플레이어가 복서를 만들고 자동 전투와 골드 강화를 통해 보스를 돌파하며, 보스 실패·오프라인·재접속 상황에서도 진행을 안전하게 이어가는 단일 성장 루프를 검증한다.

아래 v0.1은 핵심 성장 루프의 기준선이다. `수정내용2`(복서 타입·성별·스킬·복서 HP·몬스터 공격)는 TASK-004~012로 구현 완료됐고 TASK-013에서 버전·문서·테스트를 확정했다. 확장 범위는 본 문서 하단 「수정내용2 확장 범위」에 정리한다.

## 포함 기능

- 복서 생성: 이름 입력과 초기 상태 생성
- 복서 상태: 골드, 총 처치 수, 5개 전투 능력치 표시
- 자동 전투: 공격속도 기반 자동 공격, 치명타와 최근 타격 표시
- 스테이지: 장마다 일반 4개와 30초 보스 1개, 3개 테마 무한 순환
- 진행: 일반 처치 시 자동 전진, 보스 성공 시 다음 장 진입
- 파밍: 보스 실패 시 직전 일반 스테이지 반복, 수동 보스 재도전
- 강화: 골드로 공격력·공격속도·치명타율·치명타 피해·골드 보너스 강화
- 저장/불러오기: `수정내용2` 마감 확정값 `schemaVersion: 6`, `balanceVersion: 10` 진행 상태 저장
- 오프라인 보상: 최대 8시간 동안 현재 일반 스테이지 반복 파밍 정산

## v0.1 제외 기능

- 수동 공격과 클릭 데미지
- 훈련, 장비, 코치, 명성, 상대 선택, 리그와 승률 판정
- PVP, 길드, 랭킹, 시즌패스, 결제, 광고
- 서버 계정 동기화와 부정행위 방지

## 수정내용2 확장 범위

아래는 `수정내용2`로 도입 완료된 기능이다(TASK-004~012). 밸런스는 TASK-013에서 `balanceVersion: 10`·`schemaVersion: 6`으로 확정·동결했다.

- 복서 타입(인파이터/아웃복서)과 성별 외형 선택 → [복서 타입](../boxer/types.md)
- 복서 HP·몬스터 공격·회피/가드/카운터 판정 → [몬스터 공격](../combat/monster-attacks.md), [체력과 실패](../combat/hp-and-defeat.md)
- 기본 공격 4종 쿨타임·콤비네이션·그로기 → [공통 기본 공격](../combat/basic-attacks.md), [콤비네이션](../combat/combinations.md)
- 타입 전용 액티브/패시브 스킬과 장착 슬롯 → [스킬 장착 구조](../skills/equip.md)
- 게임 화면형 UI·타입별 톤·애니메이션 → [UI 구조](../presentation/ui.md)

## 완료 기준

- 신규 저장에서 복서를 생성하면 첫 공격이 한 공격 간격 뒤 자동 발생한다.
- 일반 몬스터 처치 보상이 한 번만 지급되고 다음 스테이지로 이동한다.
- 보스를 30초 안에 처치하면 다음 장으로 이동한다.
- 제한 시간 실패 시 같은 장 4스테이지에서 반복 파밍하고 버튼으로 보스를 재도전한다.
- 다섯 강화의 비용, 증가량과 상한이 기획 수식과 일치한다.
- 4장부터 1장 테마를 재사용하며 진행 수치만 장 배율로 증가한다.
- 백그라운드와 재접속에서 최대 8시간 보상이 중복 없이 정산된다.
- v6 저장은 복구되고 구버전 저장(`...v1`~`...v5`)은 삭제하지 않은 채 호환 불가(legacy) 안내 후 새 게임을 시작할 수 있다.
- 360px WebView에서 핵심 정보와 버튼이 한 열로 깨지지 않는다.

## 수용 시나리오

1. 플레이어가 복서 이름을 정하면 1-1 몬스터가 등장한다.
2. 첫 공격까지 1초를 기다린 뒤 자동으로 HP가 감소한다.
3. 몬스터를 처치해 골드를 받고 1-2로 이동한다.
4. 골드로 공격력을 강화해 피해량 증가를 확인한다.
5. 1-5 보스에 실패해 1-4 반복 파밍 상태로 돌아간다.
6. 강화 후 보스를 재도전해 처치하고 2-1로 이동한다.
7. 게임을 닫았다가 다시 열어 저장 상태와 오프라인 보상을 확인한다.
