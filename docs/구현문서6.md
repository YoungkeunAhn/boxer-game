# 작업 요청: TASK-006 게임 등급분류 자료 준비

현재 복서키우기 v0.1은 앱인토스 WebView 미니앱 출시를 목표로 개발 중이다.

이번 작업의 목표는 앱인토스 출시 전에 필요한 게임 등급분류 준비 자료를 정리하는 것이다.

이번 작업에서는 새로운 게임 기능을 추가하지 않는다.

## 이번 작업 목표

복서키우기 v0.1의 게임 등급분류 준비를 위해 필요한 설명 자료, 스크린샷 목록, 플레이 영상 계획, 기능 목록, 콘텐츠 표현 수위 설명을 정리한다.

## 작업 범위

1. 게임 기본 정보 정리
   - 게임명
   - 장르
   - 플랫폼
   - 플레이 방식
   - 핵심 루프
   - v0.1 기능 범위

2. 게임 설명 자료 작성
   - 게임 소개
   - 플레이 목적
   - 플레이 방법
   - 성장 구조
   - 경기 진행 방식
   - 보상 구조

3. 기능 목록 정리
   - 복서 생성
   - 복서 상태 보기
   - 훈련하기
   - 능력치 증가
   - 경기하기
   - 승리/패배 결과 확인
   - 보상 획득
   - 저장/불러오기
   - 재접속 시 진행도 유지

4. 콘텐츠 표현 수위 정리
   - 복싱은 스포츠 경기로 표현한다.
   - 피, 출혈, 부상, 신체 훼손 표현은 사용하지 않는다.
   - 잔혹한 KO 연출은 사용하지 않는다.
   - 타격 표현은 텍스트 또는 단순 결과 중심으로 제한한다.
   - 폭력성 수위는 낮게 유지한다.

5. 결제/광고/확률형 요소 정리
   - v0.1에는 인앱 결제가 없다.
   - v0.1에는 광고가 없다.
   - v0.1에는 유료 랜덤박스가 없다.
   - v0.1에는 확률형 유료 아이템이 없다.
   - v0.1에는 도박성 보상 구조가 없다.

6. 개인정보/계정 저장 정리
   - v0.1은 서버 저장을 사용하지 않는다.
   - v0.1은 토스 로그인 연동을 사용하지 않는다.
   - v0.1은 localStorage 기반 저장을 사용한다.
   - 개인정보 수집 여부는 공식 앱인토스 정책 확인 후 TODO로 정리한다.

7. 스크린샷 준비 목록 작성
   - 첫 화면
   - 복서 상태 화면
   - 훈련 화면
   - 경기 시작 전 화면
   - 경기 결과 화면
   - 보상 획득 화면
   - 저장 후 재접속 화면

8. 플레이 영상 촬영 계획 작성
   - 새 게임 시작
   - 복서 상태 확인
   - 훈련 실행
   - 능력치 증가 확인
   - 경기 실행
   - 승리/패배 결과 확인
   - 보상 지급 확인
   - 새로고침 후 데이터 유지 확인

## 작성 원칙

- 실제 구현된 기능만 작성한다.
- 아직 구현되지 않은 기능은 포함하지 않는다.
- 정책/법률 관련 내용은 단정하지 않고 `공식 문서 확인 필요`로 표시한다.
- 복서키우기의 폭력성은 스포츠 경기 표현 수준으로 설명한다.
- 피, 부상, 잔혹 표현이 없다는 점을 명확히 작성한다.
- 결제, 광고, 확률형 아이템, 도박성 요소가 없다는 점을 명확히 작성한다.
- 등급분류 제출 방식은 확정하지 말고 TODO로 남긴다.
