# 작업 요청: TASK-009 앱 번들 생성 및 검토 요청

현재 복서키우기 v0.1은 앱인토스 WebView 미니앱 출시를 목표로 개발 중이다.

이번 작업의 목표는 최종 빌드, 앱 번들 생성, 번들 업로드, QR 테스트, 검토 요청 준비까지 진행하는 것이다.

이번 작업에서는 새로운 게임 기능을 추가하지 않는다.

## 이번 작업 목표

복서키우기 v0.1을 앱인토스 검토 요청 가능한 번들 상태로 만든다.

## 작업 범위

1. 최종 빌드 확인
   - TypeScript 빌드 실행
   - Vite 빌드 실행
   - 빌드 에러 확인
   - 빌드 경고 확인
   - dist 산출물 확인

2. 앱인토스 설정 확인
   - granite.config.ts 확인
   - appName 확인
   - displayName 확인
   - webView type 확인
   - build command 확인
   - outdir 확인
   - icon 설정 확인
   - primaryColor 설정 확인

3. 앱 번들 생성
   - 앱인토스 공식 문서 기준으로 번들 생성
   - .ait 파일 생성 여부 확인
   - 번들 용량 확인
   - 압축 해제 기준 100MB 이하 여부 확인
   - 불필요한 리소스 포함 여부 확인

4. 콘솔 업로드 준비
   - 앱인토스 콘솔 앱 정보 확인
   - 앱 설명 입력값 확인
   - 아이콘/썸네일/스크린샷 준비 확인
   - 고객 문의 이메일 확인
   - 게임 등급분류 자료 준비 상태 확인

5. QR 테스트
   - 콘솔에서 번들 업로드
   - 토스 앱 QR 테스트 실행
   - 첫 화면 정상 표시 확인
   - 훈련 → 경기 → 보상 → 저장 루프 확인
   - 재접속 시 데이터 유지 확인
   - 모바일 화면 깨짐 여부 확인
   - Safe Area 겹침 여부 확인

6. 검토 요청 전 최종 확인
   - 실제 구현되지 않은 기능이 설명에 없는지 확인
   - 결제/광고/리더보드/친구 초대 문구가 없는지 확인
   - 피, 잔혹 표현, 유료 확률형 아이템이 없는지 확인
   - 등급분류 관련 자료가 누락되지 않았는지 확인
   - 출시 차단 이슈가 남아 있지 않은지 확인

7. 검토 요청 준비
   - QR 테스트에서 차단 이슈가 없으면 검토 요청 가능 상태로 표시
   - 차단 이슈가 있으면 검토 요청하지 않고 수정 항목으로 분리
   - 검토 요청 후 오류 발견 시 취소 후 재업로드 필요하다는 TODO 기록

- 신규 상대 추가

- 코치 시스템 확장
- 토스 로그인 연동
- 서버 저장 연동
- 인앱 결제 추가
- 광고 추가
- 리더보드 추가
- 친구 초대 추가
- 출시 후 업데이트 작업

## 완료 기준

- TypeScript 빌드가 성공한다.
- Vite 빌드가 성공한다.
- 앱인토스 번들이 생성된다.
- 번들 용량이 정책 기준을 초과하지 않는다.
- 콘솔 업로드가 가능하다.
- QR 테스트에서 v0.1 핵심 루프가 정상 동작한다.
- 검토 요청 전 차단 이슈가 없다.
- 검토 요청 가능 여부가 명확히 정리된다.
