# 작업 요청: TASK-004 앱인토스 샌드박스 실기기 테스트 및 WebView QA

현재 복서키우기 v0.1은 브라우저에서 핵심 게임 루프 구현이 진행되었고, 앱인토스 WebView SDK 연동 준비가 완료된 상태다.

이번 작업의 목표는 토스 앱인토스 샌드박스 또는 실기기 WebView 환경에서 복서키우기가 정상 실행되는지 검증하고, 모바일 WebView에서 발생하는 문제를 수정하는 것이다.

## 이번 작업 범위

이번 작업은 기능 추가가 아니라 실기기 실행 검증과 WebView QA 작업이다.

새로운 게임 시스템은 추가하지 않는다.

## 점검할 것

1. 앱인토스 실행 확인
   - 앱인토스 샌드박스 또는 토스 앱 테스트 환경에서 게임이 실행되는지 확인
   - 첫 화면이 정상 표시되는지 확인
   - 로딩 중 흰 화면이 오래 지속되지 않는지 확인
   - 콘솔 에러가 없는지 확인

2. v0.1 핵심 루프 확인
   - 복서 상태 표시
   - 훈련 실행
   - 능력치 증가
   - 전투력 갱신
   - 경기 실행
   - 승리/패배 결과 표시
   - 보상 지급
   - localStorage 저장/불러오기

3. 모바일 WebView UI 확인
   - 360px 너비에서도 화면이 깨지지 않는지 확인
   - 가로 스크롤이 생기지 않는지 확인
   - 버튼 터치 영역이 충분한지 확인
   - 하단 버튼이 safe-area에 가리지 않는지 확인
   - 긴 텍스트가 화면 밖으로 넘치지 않는지 확인

4. 앱 이탈/재진입 확인
   - 앱을 나갔다가 다시 들어왔을 때 저장 데이터가 유지되는지 확인
   - 새로고침 또는 재실행 후에도 복서 데이터가 유지되는지 확인
   - 저장 데이터가 깨졌을 때 안전하게 초기화되는지 확인

5. 앱인토스 게임 환경 대응
   - 게임 화면이 모바일 세로 모드에 적합한지 확인
   - 종료 또는 뒤로가기 상황에서 데이터 손실이 없는지 확인
   - 필요하다면 종료 확인 모달 TODO를 남긴다
   - 사운드가 없다면 v0.1에서는 사운드 없음으로 명시한다

## 수정 원칙

- 게임 로직은 변경하지 않는다.
- 수정이 필요하면 UI, 레이아웃, 저장 안정성 위주로 처리한다.
- `src/game`의 순수 함수 구조를 유지한다.
- localStorage 저장 키를 임의로 변경하지 않는다.
- WebView 전용 처리가 필요하면 별도 모듈로 분리한다.
- 해결하지 못한 항목은 TODO로 남긴다.

## 완료 기준

- 토스 앱인토스 WebView 환경에서 게임 첫 화면이 정상 표시된다.
- 훈련 → 경기 → 보상 → 저장 루프가 실기기에서 정상 동작한다.
- 앱을 나갔다가 다시 들어와도 데이터가 유지된다.
- 모바일 화면에서 핵심 UI가 깨지지 않는다.
- 출시 체크리스트 점검 단계로 넘어갈 수 있다.
