# 작업 병목 및 개선 현황

## 개요

구현 과정의 병목을 문맥 수집, 실행 환경, 검증, 기획값 관리 관점에서 추적한다. 해결 여부는 도구 존재만이 아니라 아래 완료 조건을 재현할 수 있는지를 기준으로 판단한다.

## 현황 요약

| 병목 | 상태 | 현재 대응 | 완료 조건 |
| --- | --- | --- | --- |
| 분산된 문서 구조 | 해결 | 작업별 필수 문서와 현재 구현 기준선 제공, 다중 파일 읽기 지원 | 작업 시작 시 필요한 문서를 한 명령으로 누락 없이 읽을 수 있다. |
| 의존성 설치 및 테스트 환경 | 해결 | `tools/check.mjs`가 fast·full·plan 검증을 통합 | `node tools/check.mjs full`이 테스트, 타입 검사, 빌드를 모두 통과한다. |
| 브라우저 검증 서버 연결 | 해결 | preview 주소·포트 고정, Playwright E2E(`e2e/`)가 스모크 체크리스트를 자동 검증 | `npm.cmd run e2e` 한 명령으로 핵심 루프·보스·저장/오프라인·360px를 반복 재현 검증한다. |
| 확정되지 않은 초기 밸런스 | 부분 해결 | 현재 구현값과 버전을 중앙 상수 및 기획 README에 기록 | 남은 기획값이 확정되고 플레이테스트 후 `balanceVersion`이 갱신된다. |
| Git 저장소 상태 확인 | 해결 | 저장소와 원격 브랜치 연결, 변경 경로 자동 감지 | 작업 시작 시 `git status`와 `check.mjs plan`이 변경 범위를 식별한다. |

## 표준 작업 흐름

1. `git status --short --branch`로 기존 변경을 확인한다.
2. `docs/기획/README.md`에서 작업 유형별 필수 문서를 찾는다.
3. `python tools/read-md.py "파일1" "파일2"`로 관련 문서를 한 번에 읽는다.
4. 구현 중에는 `node tools/check.mjs fast "변경 파일"`을 실행한다.
5. 설정·의존성·번들 변경 또는 최종 검증에는 `node tools/check.mjs full`을 실행한다.
6. 브라우저 검증은 빌드 후 `npm.cmd run preview:verify`와 브라우저 스모크 체크리스트를 사용한다.

## 남은 개선 과제

### 브라우저 검증 자동화 (완료)

- 고정 `0.0.0.0:4173` preview 위에서 Playwright E2E(`e2e/`)가 스모크 체크리스트를 자동 회귀 검증한다.
- 실행: `npm.cmd run e2e`(최초 1회 `npm.cmd run e2e:install`). 게이트까지 함께: `npm.cmd run check:e2e`.
- `playwright.config.ts`의 `webServer`가 `build:web → preview:verify`를 자동 기동하고, `page.clock`으로 자동 공격·보스·오프라인 타이밍을 가짜 시간으로 결정적으로 검증한다.
- E2E는 브라우저 바이너리와 실행 시간 때문에 `check.mjs full` 게이트에는 포함하지 않는다(`check:e2e`로 opt-in).
- 남은 수동 항목과 자동화 범위는 [브라우저 스모크 체크리스트](./browser-smoke-checklist.md)의 "자동화 현황"을 따른다.
- 앱인토스 WebView 검증은 SDK 연동 이후 공식 샌드박스 절차를 확인한다. (`TODO`, `공식 문서 확인 필요`)

### 밸런스 확정

- 가정: 초기 능력치, 승률 범위, 현재 훈련·상대 수치는 플레이테스트 전 기준선이다.
- `docs/기획/README.md`의 현재 구현 기준선과 `src/game/constants.ts`를 함께 갱신한다.
- 게임 규칙이나 수치를 바꿀 때 `BALANCE_VERSION`과 관련 테스트를 같이 수정한다.
- 행동력, 자동 훈련, 장비 등 미구현 시스템의 값은 확정 전까지 `TODO`로 유지한다.

## 기대 효과

- 구현 전 문맥 수집과 명령 재실행 시간을 줄인다.
- 현재 코드값과 기획 초안을 구분해 잘못된 가정으로 인한 재작업을 예방한다.
- 변경 범위에 맞는 검증과 고정된 브라우저 검증 절차를 반복 재현할 수 있다.
