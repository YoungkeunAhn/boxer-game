# 브라우저 스모크 체크리스트

## 목적

고정된 프로덕션 프리뷰에서 자동 전투, 강화, 보스, 저장·오프라인 복구와 360px 화면을 짧게 반복 검증한다. 앱인토스 WebView 샌드박스는 별도로 검증한다.

## 자동화 현황

이 체크리스트의 대부분은 Playwright E2E(`e2e/`)로 자동화돼 있다. 반복 검증은 다음 한 명령으로 재현한다.

```
npm.cmd run e2e            # chromium-desktop + mobile-360 두 프로젝트
npm.cmd run e2e:report     # 직전 실행 리포트(playwright-report/) 열기
```

- 최초 1회 `npm.cmd run e2e:install`로 Chromium을 받는다.
- E2E는 `playwright.config.ts`의 `webServer`가 `build:web → preview:verify`를 자동 실행해 `http://127.0.0.1:4173`을 띄운다. 별도 수동 기동이 필요 없다.
- 타이밍 규칙(약 1초 자동 공격, 30초 보스, 최대 8시간 오프라인)은 Playwright `page.clock`으로 가짜 시간을 제어해 실시간 대기 없이 검증한다.
- E2E는 `check.mjs full` 게이트에 포함하지 않는다(브라우저 바이너리 필요·실행 시간). 게이트까지 함께 돌리려면 `npm.cmd run check:e2e`.

아래 항목 중 **수동 유지**로 표시된 것만 사람이 직접 확인한다. 나머지는 회귀 테스트로 대체된다.

### 수동 유지 항목

- 주관적 시각 판단: 카드 밖 넘침/테마 재사용의 시각 품질, 데스크톱 읽기 순서의 자연스러움(레이아웃 좌표·테마 텍스트는 자동 검증). 전투 헤더(진행바 강조·적/복서 카드·월드맵 비활성·가로 스크롤)는 `e2e/combat-header.spec.ts`로 자동화됨 — 색상 대비·연출 품질만 수동.
- 실시간 8시간 방치(가짜 클럭 상한 테스트로 대체).
- 앱인토스 WebView 샌드박스(SDK 연동 이후 공식 절차 확인).

## 실행 준비 (수동 점검 시)

1. `node tools/check.mjs full`을 통과시킨다.
2. `npm.cmd run preview:verify`를 실행한다.
3. 브라우저에서 `http://127.0.0.1:4173`을 연다.
4. 기존 진행이 있으면 화면의 `처음부터` 기능으로 초기화한다.

검증 서버는 `0.0.0.0:4173`에 바인딩하며, 포트가 사용 중이면 다른 포트로 우회하지 않고 실패해야 한다.

## 생성과 자동 전투

- [ ] 이름을 입력해 복서를 생성하면 1-1 숲 입구가 열린다.
- [ ] 초기값이 공격력 10, 공격속도 1회/초, 치명타율 5%, 치명타 피해 2배, 골드 보너스 0%다.
- [ ] 생성 즉시 공격하지 않고 약 1초 뒤 첫 공격이 발생한다.
- [ ] 사용자가 공격 버튼을 누르지 않아도 몬스터 HP가 공격 주기마다 감소한다.
- [ ] 최근 피해와 치명타 여부에 `undefined`, `NaN`, `Infinity`가 표시되지 않는다.
- [ ] 몬스터 처치 시 골드와 총 처치 수가 한 번 증가하고 다음 스테이지로 이동한다.
- [ ] 초과 피해가 다음 몬스터 HP를 미리 줄이지 않는다.

## 강화

- [ ] 다섯 강화에 현재 값, 레벨, 다음 값과 비용이 표시된다.
- [ ] 공격력 0레벨 비용은 10골드이고 강화 후 피해가 `floor(10 × 1.2)`인 12로 증가한다.
- [ ] 골드가 부족하면 버튼이 비활성화되고 잔액이 음수가 되지 않는다.
- [ ] 공격속도를 강화하면 자동 공격 간격이 줄고 타이머가 중복 실행되지 않는다.
- [ ] 공격속도 5회/초, 치명타율 50%, 치명타 피해 5배, 골드 보너스 500%에서 상한 상태가 표시된다.
- [ ] 빠른 연속 탭으로 비용이나 레벨이 중복 반영되지 않는다.

## 스테이지와 보스

- [ ] 장마다 1~4는 일반, 5는 30초 보스로 표시된다.
- [ ] 보스 남은 시간이 감소하고 0 아래로 표시되지 않는다.
- [ ] 보스 처치 시 다음 장 1스테이지로 이동하고 보상을 한 번만 받는다.
- [ ] 보스 시간초과 시 같은 장 4스테이지 반복 파밍으로 돌아간다.
- [ ] 반복 파밍에서 일반 몬스터를 처치해도 같은 장 4에 새 몬스터가 등장한다.
- [ ] 보스 재도전 시 현재 일반 몬스터 HP를 버리고 최대 HP·30초 보스를 새로 시작한다.
- [ ] 4장은 숲 입구, 5장은 늑대 숲, 6장은 바위 협곡 테마를 재사용한다.

### 전투 헤더 (TASK-014)

- [ ] 전투 헤더의 진행바가 5칸(일반 4 + 보스 1)이며 현재 stage(1~5)와 보스 칸이 강조된다.
- [ ] `STAGE C-S` 라벨이 현재 챕터-스테이지로 표시된다.
- [ ] 몬스터 카드가 `monsterName`·현재/최대 HP·공격력(🔥)을 실시간 반영한다.
- [ ] 복서 카드가 이름·타입(라벨)·현재/최대 HP·공격력을 표시한다.
- [ ] 월드맵 버튼이 비활성(disabled)이며 클릭해도 화면 전환이 없다(동작은 TODO).

## 게임 화면형 UI (TASK-011)

- [ ] 상단 패널에 현재 스테이지(예: 1-3)·챕터 이름·복서 타입·골드·진행 상태(전투/파밍/보스전)가 표시된다.
- [ ] 중앙 전투 영역에 복서(좌)·몬스터(우)가 배치되고 복서 HP 바·몬스터 HP 바가 모두 보인다.
- [ ] 보스 진입 시 그로기 바가 나타나고, 비보스 스테이지에서는 그로기 바가 표시되지 않는다.
- [ ] 그로기 게이지가 상한에 도달하면 그로기 상태 강조가 보인다(텍스트 "그로기!").
- [ ] 데미지 숫자·콤보명·스킬명·MISS/GUARD/COUNTER 텍스트가 전투 중 표시되고 `undefined`/`NaN`/`Infinity`가 없다.
- [ ] 하단 패널에 기본 공격 4종(잽/스트레이트/훅/어퍼) 쿨타임 진행도가 각각 표시되고, 준비되면 "준비됨"으로 바뀐다.
- [ ] 하단 패널에 전용 스킬 슬롯 1~3 + 패시브가 표시되고 장착/해제 버튼이 동작한다(표시명 노출).
- [ ] 강화에 기존 5종(공격력·공격속도·치명타율·치명타 피해·골드 보너스) + 신규 4종(체력·방어·회피·카운터)이 모두 표시되고, 잔액 부족 시 비활성·상한 도달 시 MAX가 표시된다.

## 타입별 톤·애니메이션 (TASK-012)

- [ ] 공격 4종(잽/스트레이트/훅/어퍼)·회피·피격·사망·스킬 모션이 결과에 맞춰 복서 아바타에 적용된다(`data-attack-key`로 자동 검증).
- [ ] 인파이터/아웃복서가 시각적으로 구분된다(루트 `data-boxer-type`·톤 클래스, 자동 검증).
- [ ] 인파이터 톤: 묵직한 타격·아레나 화면 흔들림·붉은 압박감·GUARD 강조(수동 시각 판단).
- [ ] 아웃복서 톤: 잔상·빠른 스텝·MISS/COUNTER 강조(수동 시각 판단).
- [ ] 전투 텍스트(데미지·콤보·스킬명·MISS/GUARD/COUNTER)가 팝/페이드로 등장하고 연속 동일 타격에도 재시동된다.
- [ ] 연출 추가 후에도 핵심 정보(복서/몬스터 HP·feed·타이머)가 360px 안에 보이고 가로 스크롤이 없다(자동 검증).
- [ ] 60fps 목표: transform/opacity 기반 가벼운 연출만 쓰고 끊김이 없다(수동 시각 판단).
- [ ] `prefers-reduced-motion: reduce`에서 애니메이션이 비활성화돼도 핵심 정보가 그대로 보인다(자동 검증).
- [ ] 등급분류: 출혈·부상·잔혹·사망 강조 표현이 없다(수동 점검; 사망은 절제된 "쓰러짐"만).

## 저장, 백그라운드와 오프라인 보상

- [ ] 새로고침 후 이름, 골드, 총 처치 수, 강화 레벨과 위치가 복원된다.
- [ ] 새로고침 시 몬스터는 현재 HP가 아니라 최대 HP에서 새로 시작한다.
- [ ] 탭을 백그라운드로 보내면 온라인 공격이 멈춘다.
- [ ] 복귀 시 현재 일반 스테이지 반복 파밍 보상 요약이 한 번만 표시된다.
- [ ] 보스 중 백그라운드 진입 후 복귀하면 같은 장 4스테이지로 돌아간다.
- [ ] 8시간을 넘는 이탈도 최대 8시간까지만 인정된다.
- [ ] v1 저장이 있으면 삭제하지 않고 호환 불가와 새 게임 안내가 표시된다.
- [ ] 손상되거나 지원하지 않는 v2 저장은 오류 없이 새 게임 화면을 표시한다.
- [ ] 저장소 쓰기 실패 시 현재 세션은 유지되고 새로고침 시 유실 가능성이 안내된다.

## 360px 모바일 화면

- [ ] viewport를 360px 너비로 설정해도 가로 스크롤이 생기지 않는다.
- [ ] 상태, 자동 전투, 강화 영역이 한 열로 배치된다.
- [ ] 모든 버튼과 입력 필드가 최소 44px 터치 영역을 가진다.
- [ ] 긴 복서 이름, 큰 골드·처치 수와 결과 메시지가 카드 밖으로 넘치지 않는다.
- [ ] 복서 HP 바·몬스터 HP 바·(보스)그로기 바·기본 스킬 쿨타임 바가 360px 안에 들어오고 잘리지 않는다.
- [ ] 상단/중앙/하단 3영역이 한 열로 세로 적층되고 가로 스크롤이 없다.
- [ ] 초기화 확인 창을 취소하면 진행도가 유지된다.

## 데스크톱과 장시간 실행

- [ ] 데스크톱 너비에서도 카드 간격과 읽기 순서가 자연스럽다.
- [ ] 여러 번의 강화와 보스 재도전 후에도 공격 한 번당 HP가 한 번만 감소한다.
- [ ] 장시간 방치 후 보스 실패·파밍 전이가 한 번만 발생한다.
- [ ] 콘솔에 React 키, 상태 업데이트, 타이머와 저장 관련 오류가 없다.

## 완료 기록

- 검증 일시, 브라우저·viewport, 대상 커밋을 작업 결과에 기록한다.
- 실패 항목은 재현 절차와 함께 이슈 또는 작업 문서에 남긴다.
- 자동 전투 간격과 보스 경계 시각처럼 수동 검증이 어려운 규칙은 가짜 타이머 자동 테스트 결과를 함께 확인한다.
