# 공통 레이아웃 — 상단 바 · 하단 네비 · 재화

모든 화면에 공통으로 깔리는 프레임. 출처 이미지: `메인ui1.png`, `메인ui2.png`(전 화면에 동일 상단/하단 반복).

## 1. 이미지에서 확인된 요소

### 상단 바 (모든 화면 공통)

```text
[프로필 아바타 | 복서키우기 | Lv.45 | 빨강 경험치 바 72%]   [🪙 128.4K] [💎 2,350] [⚡ 35/50] [⏱ 02:15]
```

- 좌측: 원형 프로필 아바타, 게임/플레이어 이름, **플레이어 레벨(Lv.45)**, 그 아래 **경험치 진행 바(72%)**.
- 우측 재화 4종:
  - 🪙 **골드** (128.4K) — 기존 재화.
  - 💎 **다이아(유료/프리미엄 재화)** (2,350) — 신규.
  - ⚡ **에너지/스태미나** (35/50) — 신규, 상한 있음.
  - ⏱ **타이머** (02:15) — `가정:` 에너지 1 회복까지 남은 시간 또는 일일 리셋 카운트다운.

### 하단 네비게이션 (5탭, 모든 화면 공통)

```text
[상점] [가방] [파이터] [퀘스트] [경기장]
  🛒    🎒    🥊(중앙)   📋     🥊링
```

- 중앙 **파이터**가 기본/홈 탭(메인 전투 화면). 선택 탭은 강조(빨강) + 라벨.
- 탭 아이콘 우상단에 **빨강 알림 뱃지(!)** — 수령 가능한 보상/신규가 있을 때(상점·가방·퀘스트·경기장에 표시됨).

## 2. 현재 상태

- 재화: **골드 단일**(`boxer.gold`). 다이아·에너지·플레이어 레벨/경험치 **없음**.
- 화면: **단일 화면**(`HomePage`) 구조. 탭 네비게이션·라우팅 **없음**.
- 상단 요약은 [UI 구조 §상단 UI](../presentation/ui.md)의 `BoxerStatus`가 일부 담당(이름/타입/골드/스테이지/능력치). 플레이어 레벨·다중 재화·타이머는 없음.

## 3. 작성 내용 (결정 반영)

> 결정(2026-06-21): **다이아·플레이어 레벨/경험치 채택, 에너지 미채택.** 상점은 무과금만, 경기장은 구현 보류. 미확정 수치는 `가정:`/`TODO`로 표기.

### 3-1. 재화 모델 (확정 방향)
- **골드(🪙)** — 기존 `boxer.gold` 유지. 처치 보상·강화 비용의 기축 재화.
- **다이아(💎, 프리미엄 재화)** — 채택. **무과금 획득만**(실결제 없음):
  - 획득처: 퀘스트·업적 보상, 일일 무료 상자, 마일스톤 상자([05-퀘스트](./05-퀘스트.md)). 결제 획득은 보류([03-상점](./03-상점.md)).
  - 사용처: [03-상점](./03-상점.md) 무과금 상품 구매. 강화/뽑기 연동은 `TODO`(장비·경기장 보류와 함께 후순위).
  - 단위: 정수, `MAX_SAFE_GAME_INTEGER` 클램프.
- **에너지(⚡) — 미채택(보류).** 상단 바에서 제외. 도입한다면 [06-경기장](./06-경기장-리그.md)·특수 콘텐츠 도전 전용(본편 방치 전투는 무제한)이나, 경기장 자체가 구현 보류이므로 현재 단계에서 제외 → `Energy` 저장 필드·오프라인 회복 로직 **불필요**.
- **타이머(⏱) 재해석.** 에너지 미도입이므로 에너지 회복 카운트다운이 아니라 **일일 콘텐츠 리셋까지 남은 시간**(다음 로컬 00:00)으로 정의. 주입 `now` 기준 순수 파생([05-퀘스트](./05-퀘스트.md) 일일 리셋과 동일 기준).
- **플레이어 레벨/경험치(Lv.45 + 경험치 바)** — 채택. **전투 강화 레벨과 별개**:
  - 획득원(`가정:`): 몬스터 처치·보스 클리어·퀘스트 완료. (오프라인 처치수로도 파생 가능 — [오프라인 정산](../systems/stage-and-offline.md)이 이미 처치수를 계산)
  - 레벨업 보상: `TODO`(다이아 소량? 능력치 보정? 밸런스 확정 전 미정).
  - 곡선: 아래 §3-5 `가정:`.

### 3-2. 레이아웃/네비게이션
- **5탭, 단일 페이지 내 탭 전환**(`가정:` 라우터 미도입 — 현 `HomePage`에 탭 상태만 추가). 중앙 **파이터**가 홈/기본 탭.
- **경기장 탭은 구현 보류**([06](./06-경기장-리그.md)) → 초기 릴리스에서는 **숨김 또는 비활성(잠금 표시)**(`가정:`). 5탭 자리는 유지하되 진입 차단.
- **알림 뱃지 규칙**: 해당 탭에 *수령 가능한* 보상/신규가 있을 때만 빨강 점.
  - 상점: 무료 상자 수령 가능 시.
  - 퀘스트: 완료·미수령 퀘스트 또는 마일스톤 상자 수령 가능 시([05](./05-퀘스트.md)).
  - 가방·경기장: 보류 → 뱃지 없음.
- **360px 배치**: 재화가 골드·다이아 2종 + 일일 타이머로 단순화 → 좌측 프로필·이름·레벨바, 우측 `[🪙골드][💎다이아][⏱일일리셋]`. 4재화 → 2재화로 가로 압박 완화.
- **월드맵 버튼**: 파이터 화면 상단([02-파이터](./02-파이터-메인화면.md) §3-1).

### 3-3. 데이터/저장 영향
- `SaveData` 추가 필드(에너지 제외): `diamond: number`, `playerLevel: number`, `playerExp: number`. (`expToNext`는 저장하지 않고 순수 함수 파생 — 강화 레벨과 동일 패턴)
- **`SCHEMA_VERSION` 1회 범프**(README Phase 표: P3에서 v6→v7) + 저장 키 접미사 일치(`boxer-game.save.vN`). 옛 저장은 삭제 말고 `legacy`/`invalid` 안내.
- 타입가드 추가: `diamond`/`playerLevel`/`playerExp`는 유한·음수불가·세이프정수 범위 검증([save.ts](../../src/game/save.ts) 패턴).
- **오프라인 처리**: 에너지 없음 → 에너지 회복 정산 불필요. 일일 리셋 타이머·경험치는 주입 `now`/처치수 기준 순수 파생([오프라인](../systems/stage-and-offline.md)과 일관).

## 3-5. 데이터 모델 초안 (결정 반영 — 미확정 수치만 `가정:`)
```ts
// SaveData 확장 (P3) — 에너지 제외
type Wallet = {
  gold: number          // 기존 boxer.gold (지갑 이동은 선택 — 현행 위치 유지 가능)
  diamond: number       // 신규 프리미엄 재화 (무과금 획득)
}

type PlayerProgress = {
  level: number         // Lv.45 (전투 강화 레벨과 별개)
  exp: number           // 현재 경험치
  // expToNext는 저장하지 않고 순수 함수로 파생
}

// 에너지: 미채택 — 도입 시에만 추가(경기장 전용), 현재 단계 제외
// type Energy = { current; max; updatedAt }   // 보류
```

### 일일 리셋 타이머 (에너지 회복 대체 — `가정:`)
```text
dailyResetAt = 다음 로컬 00:00 (epoch ms)
표시 타이머 = dailyResetAt - now      // ⏱02:15 위치에 일일 리셋 카운트다운
now >= dailyResetAt 이면 일일 퀘스트/무료상자/상점 일일상품 초기화 ([05]·[03]와 동일 기준)
```
- 주입 `now` 기준 순수 함수. `Date.now` 직접 사용 금지(프로젝트 규칙).

### 플레이어 경험치 곡선 예시 (`가정:`)
```text
expToNext(level) = floor(BASE_EXP * GROWTH^level)   // 가정: 기존 1.25^ 비용 곡선과 톤 맞춤
획득원: 몬스터 처치 / 보스 클리어 / 퀘스트  (수치 TODO)
레벨업 보상: TODO (다이아 소량 등 — 밸런스 확정 전 미정)
```

## 4. 범위·커플링 플래그
