우측 능력치 요약: 🔥공격력 3.25K / ❤체력 8,450 / 👟속도 1.32 / ☀치명타 12.5%  [상세 정보 >]

- 착용 슬롯 약 6개(헤드기어·글러브·바디·신발·하의·액세서리 추정).

## 2. 현재 상태

- **장비 시스템 없음.** 아이템·인벤토리·등급·강화·착용 슬롯 개념 전무.
- [MVP 범위](../overview/mvp-scope.md): **장비 명시적 제외**.
- **결정(2026-06-21): 장비·훈련 시스템 보류(P4).** [02-파이터](./02-파이터-메인화면.md) 강화 패널은 **현행 9종 유지**(장비강화 축 미채택)로 확정 → 두 화면의 커플링이 해소됨. 따라서 본 문서는 단독으로 보류 가능(02를 막지 않음). 아래 데이터 모델 초안은 **P4 도입 시 출발점**으로 유지.

### 3-1. 아이템 데이터 모델
- [ ] 아이템 타입: 장비/소모품/재료/기타 4분류 정의.

- [ ] 아이템 속성: id, 슬롯, 등급, 레벨, 강화단계(+N), 기본 옵션, 추가 효과(랜덤 옵션?), 스택 가능 여부.

### 3-2. 능력치 반영
- [ ] 장비가 더하는 능력치(공격력·체력·속도·치명타 등)가 [능력치와 수식](../systems/stats-and-formulas.md)의 최종 계산에 합산되는 방식.

### 3-3. 강화/획득

- [ ] 획득처: 드랍/상자([상점](./03-상점.md))/제작(재료 탭).

## 3-5. 데이터 모델 초안 (사용자 확정용 — `가정:`)

```ts
// 가정: 장비 아이템 (P4)
type ItemGrade = 'common' | 'rare' | 'epic' | 'legendary'   // 이미지: E/희귀… 등급명 TODO
type EquipSlot = 'headgear' | 'gloves' | 'body' | 'shoes' | 'bottom' | 'accessory'

type EquipmentItem = {
  id: string            // 고유 인스턴스 id
  defId: string         // 아이템 정의 id (이름·기본옵션 출처)
  slot: EquipSlot
  grade: ItemGrade
  level: number         // 이미지 Lv.15
  enhance: number       // 강화단계 +N
  // 추가 효과(랜덤 옵션)는 정의에서 파생 or 인스턴스에 저장 — TODO
}

type Inventory = {
  items: EquipmentItem[]              // 보유 목록 (이미지 48/120)
  capacity: number                    // 120
  equipped: Partial<Record<EquipSlot, string>>  // slot → item.id
  // 소모품/재료는 스택 카운트 맵으로 별도: materials: Record<string, number>
}
```

### 능력치 합산 위치 (`가정:`)
```text
calculateCombatStats(boxer)
  = 기본 강화 스탯
  + Σ 착용 장비의 옵션(공격력·체력·속도·치명타…)
  × (1 + 0.01 * 장비강화레벨)   // 02-파이터 "장비강화" 글로벌 배수
```
- 이 변경은 `calculateCombatStats`를 건드리므로 전투/콤비네이션 테스트 전부 재확인 + `BALANCE_VERSION`.

### 장비 강화 수식 예시 (`가정:`)
```text
enhanceCost(enhance) = floor(BASE * 1.25^enhance)        // 골드
성공률·파괴: TODO (MVP는 100% 성공·비파괴 권장)
옵션 증가 = 기본옵션 * (1 + 0.1 * enhance)               // 가정
```
