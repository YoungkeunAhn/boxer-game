## 3. 작성 내용 (구현 보류 — 도입 시 채울 섹션)

> 결정(2026-06-21): **경기장(PVP) 구현 보류.** 서버가 없어 진짜 동기 대인전은 불가하며, 현재 단계에서는 구현하지 않는다. 아래는 도입(P5, 최후순위) 시 출발점으로 유지한다. 도입한다면 README가 권장한 **"더미 상대 + 로컬 시뮬"** 형태가 유일하게 실현 가능한 방식이다. 또한 [01-공통](./01-공통-레이아웃.md)에서 **에너지를 미채택**했으므로, 도입 시 도전 비용(⚡) 부분은 에너지 도입 결정과 함께 재설계해야 한다(현재는 에너지 없음). 미확정 수치는 `가정:`/`TODO`.

## 3-5. 데이터 모델 초안 (사용자 확정용 — `가정:`)

```ts
// 가정: 경기장 (P5) — 서버 부재 → 비동기/더미 상대 + 로컬 시뮬
type LeagueTier = 'bronze' | 'silver' | 'gold' | 'platinum' | 'diamond'

type ArenaState = {
  trophies: number          // 1,285
  tier: LeagueTier
  tierStep: number          // 실버 II → 2
  season: number            // 시즌 12
  seasonEndsAt: number      // 카운트다운 기준
  defenseRecord: { win: number; lose: number }   // 32승 15패
  weeklyWins: number        // 42 (주간 보상 마일스톤)
  weeklyRewardClaimed: number[]
  opponents: OpponentSnapshot[]   // 현재 제시된 3명
  refreshLeft: number       // 무료 갱신 3/3
  history: BattleRecord[]    // 최근 전투
}

type OpponentSnapshot = {
  name: string; tier: string; trophies: number; power: number
  winRate: number; recent10: number
  // 더미 생성: 내 트로피 ± 범위에서 능력치 스냅샷 합성
}
```

### PVP 판정 방식 결정 (핵심 — `가정:`)
```text
방식 A (권장, 무서버): 더미 상대 능력치 스냅샷 vs 내 calculateCombatStats →
  기존 전투 로직 간이 시뮬(주입 random) → 승패 → 트로피 증감
방식 B: 별도 간이 승패 공식(power 비교 + 난수)
트로피 증감 예시: 승 +N(상대 트로피 높을수록 ↑), 패 -M (가정)
```
- 서버 없는 한 "내 캐릭터가 상대 더미를 자동 격파" 형태만 가능. 진짜 대인 동기 매칭은 불가 — 사용자 결정 필요.

### 시즌/순위 (`가정:`)
```text
순위(1,238위)는 서버 없으면 실제 산출 불가 → 더미 분포로 추정 표시 or 기능 보류.
시즌 리셋: seasonEndsAt 경과 시 트로피 일부 감산 + 보상 지급 (주입 now 기준)
```

## 4. 범위·커플링 플래그

- ⏸ **구현 보류(P5, 최후순위).** 현재 미구현, 문서만 유지. 아래는 도입 시 파급도.
- 🔴 **MVP 최대 충돌(PVP·랭킹·서버).** 서버 동기화 없는 현재 구조에서 진짜 PVP는 불가 → "더미 상대 + 로컬 시뮬" 형태로만 가능. 도입 시 mvp-scope 전면 재정의.
- 🔴 트로피·시즌·전적 = 저장 스키마 추가 → `SCHEMA_VERSION`.
- 🔴 PVP 전투 판정 = 신규 순수 로직 모듈 + `BALANCE_VERSION`.
- 🟡 Phase상 **가장 후순위 권장**(상점과 함께 마지막 단계).
