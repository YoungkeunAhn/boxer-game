# 캐릭터 타입별 애니메이션 (스프라이트 시트)

출처 이미지: `인파이터_남자.png`, `인파이터_여자.png`, `아웃파이터_남자.png`, `아웃파이터_여자.png`. 각 시트에 **6포즈**가 명시됨.

## 1. 이미지에서 확인된 요소

### 공통
- 타입/성별 4종 각각 별도 스프라이트(자세·실루엣·색 톤 차이).
- **인파이터 = 붉은 톤 / 아웃파이터 = 푸른 톤**(트렁크·이펙트 색).
- 부제(키워드)로 타입 성향을 명시.

### 인파이터 (남/여) — "근접 압박형 · 강력 근접 · 카운터 특화"
```text
1. 기본 가드   2. 타이트 가드   3. 숏 잽
4. 바디 훅     5. 어퍼컷       6. 카운터 훅
```

### 아웃파이터 (남/여) — "거리 조절형 · 빠른 스텝 · 카운터 중심"
```text
1. 롱 가드     2. 라이트 풋워크   3. 스텝 백 (회피)
4. 롱 잽       5. 스트레이트 크로스  6. 카운터 펀치
```
- 인파이터 여자 부제는 "인파이팅 중심", 나머지는 "카운터 중심" — 미세 차이 확인 필요.

## 2. 현재 상태

- [애니메이션 요구사항](../presentation/animation.md)에 **공통 모션(대기/공격/피격/회피/사망/스킬)**, 공격 4종(잽·스트레이트·훅·어퍼), 회피 6종, 애니키 규칙이 이미 정의됨.
- [타입별 UI 톤](../presentation/ui-tone.md)에 인파이터=붉은/묵직, 아웃복서=푸른/잔상 톤 정의됨.
- [캐릭터 성별](../boxer/gender.md)에 외형·모션 무게감 차이 정의(추정).
- **갭**: 이미지가 제시한 **타입별 고유 6포즈 명칭**과 현재 모션 목록의 매핑이 문서에 없음. 예: 인파이터의 "타이트 가드/바디 훅/카운터 훅", 아웃파이터의 "라이트 풋워크/스텝 백/스트레이트 크로스/카운터 펀치"는 일반 공격 4종과 별개의 **타입 전용 포즈**.

## 3. 작성 내용 (결정·코드 반영)

> 결정(2026-06-21): [02-파이터](./02-파이터-메인화면.md) 타입 전환은 **단일 캐릭터 외형 전환** → 4종(타입×성별) 스프라이트는 모두 **같은 캐릭터의 교체 가능한 외형**이다(별개 4캐릭터 아님). 런타임 전환 시 외형·모션·타입 전용 스킬이 함께 바뀐다. 미확정(아트 파이프라인) 수치는 `가정:`/`TODO`.

### 3-1. 포즈 ↔ 게임 상태 매핑 (확정 키 기준)
[애니메이션 키](../presentation/animation.md)에서 확인된 키로 매핑(공격 4종 `boxer_left_jab`·`boxer_right_straight`·`boxer_left_hook`/`boxer_right_hook`·`boxer_left_upper`/`boxer_right_upper`, 회피 6종 더킹·위빙·스웨이·스텝백·고스트스텝·나비스텝):
- 가드(기본/타이트/롱) → 대기 `boxer_idle`(평상시) + 피격 방어 시 가드 표시([GUARD 판정](../combat/hp-and-defeat.md)). "타이트/롱"은 타입별 가드 외형 변형.
- 라이트 풋워크 → 대기(idle) 변형(아웃파이터의 가벼운 스텝 idle).
- 스텝 백(회피) → 회피 판정([몬스터 공격](../combat/monster-attacks.md)) 시. 기존 회피 6종의 **스텝백** 키 재사용(신규 키 아님).
- 숏 잽/롱 잽/바디 훅/어퍼컷/스트레이트 크로스 → 기본 공격 4종(`boxer_*_jab`/`*_hook`/`*_upper`/`*_straight`)에 대응.
- 카운터 훅 / 카운터 펀치 → 카운터 발동 시. **현재 전용 카운터 키 없음 → `boxer_counter` 신규 키 필요**(카운터는 메커니즘만 존재, 명명 애니 미정의).

### 3-2. 타입별 차별화 (단일 캐릭터 외형 전환)
- 같은 "잽"이라도 인파이터=**숏 잽**(근접·짧은 리치), 아웃파이터=**롱 잽**(거리·긴 리치): 동일 공격 키를 타입 외형으로 길이/이동 거리만 다르게 연출.
- 인파이터 6포즈/아웃파이터 6포즈는 [타입별 스킬](../skills/infighter-skills.md)·[아웃복서 스킬](../skills/out-boxer-skills.md) 및 판정(가드/회피/카운터)에 연결해 타입 정체성 강화. 4종이 한 캐릭터의 외형이므로 **전환 시 6포즈 세트 전체가 교체**된다.
- 성별 무게감 차이는 [성별 문서](../boxer/gender.md) 정의(인파 남=묵직, 인파 여=빠른 압박, 아웃 남=긴 스트레이트·카운터, 아웃 여=고스트/나비 스텝)를 모션 톤에 반영.

## 3-4. 포즈 매핑 (코드 확정 — TASK-018, `src/game/animation.ts` `POSE_MAP`)

> 이미지 6포즈를 [기존 모션 목록](../presentation/animation.md)·전투 판정에 잇는 매핑. 아래 표는 코드의 `POSE_MAP`과 1:1로 일치한다(테스트 `src/game/animation.test.ts`가 표 일치를 단언).

| # | 인파이터 | 아웃파이터 | 재생 시점 | 인파 키 / 아웃 키 | 리치 |
| --- | --- | --- | --- | --- | --- |
| 1 | 기본 가드 | 롱 가드 | idle(대기) | `boxer_idle` / `boxer_idle` | 인파 SHORT / 아웃 LONG |
| 2 | 타이트 가드 | 라이트 풋워크 | 가드(GUARD 판정) / 대기변형 | `boxer_guard` / `boxer_guard` | 인파 SHORT / 아웃 LONG |
| 3 | 숏 잽 | 스텝 백(회피) | 인파=잽 / 아웃=회피판정 | `boxer_left_jab` / `boxer_dodge` | 인파 SHORT / 아웃 LONG |
| 4 | 바디 훅 | 롱 잽 | 훅 / 잽 | `boxer_left_hook` / `boxer_left_jab` | 인파 SHORT / 아웃 LONG |
| 5 | 어퍼컷 | 스트레이트 크로스 | 어퍼 / 스트레이트 | `boxer_left_upper`(좌/우 hand) / `boxer_right_straight` | 인파 SHORT / 아웃 LONG |
| 6 | 카운터 훅 | 카운터 펀치 | 카운터 발동 시 | `boxer_counter` **(신규 키)** | 인파 SHORT / 아웃 LONG |

- **신규 애니 키는 `boxer_counter` 하나뿐**(POSE_6). 나머지 5포즈는 모두 기존 키 재사용.
- **카운터 발동 트리거 확정**: 애니 키 도출(`resolveAnimationKey`)은 우선순위 **카운터(`recentDefense.outcome==='COUNTER'`) > 회피(`MISS` → `boxer_dodge`) > 가드(`GUARD` → `boxer_guard`) > 직전 공격 > idle**. 즉 `boxer_counter`는 아웃복서가 회피에 성공해 카운터가 터진 경우(`outcome==='COUNTER'`)에만 재생한다. 인파이터의 가드 반격(`counterDamage>0`이지만 `outcome==='GUARD'`)은 가드 모션으로 분류한다(카운터 애니 아님).
- **회피 중복 정리(결론·재사용)**: 아웃파이터 "스텝 백/라이트 풋워크"는 신규 키를 만들지 않고 기존 키를 재사용한다 — 스텝 백 = `boxer_dodge`(회피 6종의 스텝백), 라이트 풋워크 = `boxer_guard`(대기변형). 회피 6종(더킹/위빙/스웨이/스텝백/고스트스텝/나비스텝) 중 현재 도출 계층은 회피 결과를 단일 `boxer_dodge`로 매핑한다(세분화는 TODO).
- 타입 정체성: 같은 키라도 인파이터=숏(SHORT, 근접), 아웃파이터=롱(LONG, 거리) — `POSE_MAP`의 `reach`로 표현하고 CSS(`BoxerFigure.module.css`)가 이동 거리/스케일을 차별화한다. 톤은 [`TYPE_TONE`](../기획/presentation/ui-tone.md): 인파 적/`shake`, 아웃 청/`afterimage`.
