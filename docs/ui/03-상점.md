## 1. 이미지에서 확인된 요소

### 탭
```text

```

### 추천 탭 상품 목록
- **스타터 패키지** (배너형): "초보자를 위한 최고의 선택", 50% 할인, 구성 = 💎1,200 + 🪙100K + 🥊x10 + 상자x1, ₩11,000 → **₩5,900**(실결제).
- **무료 상자**: 매일 1회 무료, [무료 받기] 버튼.
- **일일 보급품** (HOT): 💎100 + 🪙10K + 🥊x5, 매일 한정 1회, 🪙100.
- **에너지 충전**: ⚡x50, 즉시 충전, 💎80.
- **골드 꾸러미** (BEST): 50,000 골드, 🪙50K.
- **다이아 팩** (BEST): 1,000 다이아, ₩5,900(실결제).
- **훈련 티켓** (NEW): 훈련 10회 즉시 완료, 💎300.
- **장비 상자**: 장비 1개 확정, 💎200.
- **스킬 코어**: 스킬 강화 재료, 🪙150.
- **스테미나 충전**: 스테미나 100 충전, ⚡(아이콘) — `가정:` 다이아 결제.
- **오늘의 한정** (-40%, 카운트다운 07:44:28): 전설 글러브 조각 x20, 💎500.
- 하단 주의문: "상품 구매 시 청약철회가 제한됩니다." + [확률 정보] 링크.

### 결제/재화 종류
- **실결제(₩)**, **다이아(💎)**, **골드(🪙)**, **에너지/스태미나(⚡)** 혼합.
- 뱃지: BEST / HOT / NEW / 할인%, 카운트다운 한정.

## 2. 현재 상태

- **상점 없음.** 구매·결제·다이아·에너지·상자(가챠)·무료 보상 개념 전무.
- [MVP 범위](../overview/mvp-scope.md)는 **결제·광고·장비·훈련 명시적 제외**. 본 화면은 그 전부에 의존.

## 3. 작성 내용 (결정 반영)

> 결정(2026-06-21): **무과금 상점만.** 실결제(₩) 상품·UI 보류, 통화는 **골드·다이아만**. 에너지·장비·훈련 의존 상품은 해당 시스템 보류로 함께 보류. 미확정 수치는 `가정:`/`TODO`.

### 3-1. 결제/정책 (확정)
- **인앱결제(IAP) 미도입.** 앱인토스 v0.1 제외 범위([플랫폼](../release/platform-apps-in-toss.md): "인앱 결제" 제외). 상점은 **무과금**(골드·다이아)만.
- 결제가 없으므로 **영수증 검증·서버 불필요**, 청약철회 고지 N/A. (MVP "서버 동기화 제외"와 정합)
- 확률형 상자를 무과금으로라도 도입하면 **확률 고지 검토 필요** — [게임 등급](../release/game-rating.md)은 "추후 결제·광고·랜덤 보상 추가 시 등급·고지·청소년 보호 요건 재검토"라고 명시. 현재는 확률형 상자도 보류 권장(§3-3).

### 3-2. 상품 카탈로그 (무과금 실현 범위)
- 통화는 **골드(🪙)·다이아(💎)** 만. `krw`(실결제)·`energy`(에너지) 제외.
- **현재 실현 가능한 상품**(번들·환전·무료):
  - 무료 상자(매일 1회) — 골드·다이아 소량(`가정:` 정액 지급 권장, 확률형 아님).
  - 다이아 → 골드 환전 꾸러미(다이아로 골드 구매).
  - 골드/다이아 번들(퀘스트·업적 보상 연계 — 무료 획득 다이아의 사용처).
- **보류 상품**(전제 시스템 미도입): 스타터/다이아 팩(실결제), 에너지·스태미나 충전(에너지 미채택), 훈련 티켓(훈련 보류), 장비 상자·전설 글러브 조각(장비 보류), 스킬 코어(스킬강화 축 미도입). → 해당 시스템 도입 시 추가.
- **일일 무료/한정 리셋**: [01-공통 일일 리셋 타이머](./01-공통-레이아웃.md)(다음 로컬 00:00)와 동일 기준. 구매/수령 횟수는 저장 추적.
- "오늘의 한정" 카운트다운: 주입 `now` 기준 순수 파생(`Date.now` 금지).

### 3-3. 확률형/상자
- **현 단계 확률형(가챠) 보류.** 장비·스킬강화 축이 보류라 드랍 대상 자체가 없음. 무료 상자는 **확률형이 아닌 정액 지급**으로 설계.
- 추후 확률형 도입 시 [확률 정보] 화면 + 가중 확률 고지 필수([게임 등급](../release/game-rating.md) 재검토 트리거).

### 3-4. 재화 흐름
- 지급 재화는 [01-공통 재화 모델](./01-공통-레이아웃.md)(골드·다이아)과 일치. 실결제·에너지 항목 없음.
- 경계: **골드 상품**(소액·환전 결과) vs **다이아 상품**(프리미엄 — 단 무과금 획득). 실결제 경계는 없음(보류).
- 상점은 무료 획득 다이아의 주요 **사용처(sink)** 역할.

## 3-5. 데이터 모델 초안 (결정 반영 — 무과금)

```ts
// 상품 카탈로그 (무과금) — krw·energy 제외
type Currency = 'diamond' | 'gold'
type ShopTab = 'recommend' | 'currency' | 'special'   // 'equip' 탭은 장비 보류로 제외
type ShopBadge = 'best' | 'hot' | 'new' | 'sale'

type ShopProduct = {
  id: string
  tab: ShopTab
  price: { currency: Currency; amount: number }   // 무과금 통화만
  contents: Reward[]            // 번들 구성 (골드·다이아) — 아이템/상자는 보류
  badge?: ShopBadge
  discountPct?: number
  limit?: { kind: 'daily' | 'oneTime' | 'timed'; resetAt?: number }
  free?: boolean                // 무료 상자
}

// 저장: 구매/수령 추적
type ShopState = {
  purchasedCount: Record<string, number>   // productId → 횟수 (일일/한정 검증)
  dailyResetAt: number                      // 01-공통 일일 리셋과 동일 기준
}
```

## 4. 범위·커플링 플래그

- 🟢 **무과금만** → 결제 심의·서버·영수증 이슈 없음(범위 대폭 축소). 실결제 상품은 보류 트랙.
- 🟡 무과금 상점 = 다이아·골드 sink. 구매/수령 상태(`ShopState`)는 저장 스키마 추가 → `SCHEMA_VERSION`(01-공통과 같은 P3 범프에 합류 가능).
- 🟢 확률형 보류 → 확률 고지 의무 현 단계 미발생. 도입 시 [게임 등급](../release/game-rating.md) 재검토 트리거.
- 🟡 일일/한정/카운트다운은 순수 로직(`now` 주입)로 — `Date.now` 직접 금지.
