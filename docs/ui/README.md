## 가장 큰 발견: 예시 UI는 현재 MVP 범위를 크게 초과한다

현재 [MVP 범위](../overview/mvp-scope.md)는 **단일 화면 / 골드 단일 재화 / 5+4 강화 / 타입·성별·스킬**까지다. 다음은 명시적으로 **제외** 항목이다: 훈련, 장비, 명성, 상대 선택, 리그·승률, PVP, 길드, 랭킹, 시즌패스, **결제, 광고**.

예시 이미지는 아래를 새로 요구한다 — 즉, 대부분이 **MVP 경계 밖**이다. 진행 전 사용자 결정이 필요하다(아래 "범위 결정 필요" 참조).

| 예시 화면 | 새 기능 | 현재 문서/구현 | MVP 상태 |
| --- | --- | --- | --- |
| 공통 상단/하단 | 다중 재화(다이아·에너지), 플레이어 레벨, 5탭 네비, 월드맵 | 없음(골드 단일, 단일 화면) | 🔴 범위 밖 |

| 상점 | IAP 패키지, 재화 번들, 에너지 충전, 일일 무료/한정 | 없음 | 🔴 결제 — 범위 밖 |
| 가방 | 장비 슬롯·아이템·등급·강화 | 없음(MVP 제외) | 🔴 장비 — 범위 밖 |

## 도입 전략: Phase별 확장 (확정)

한 번에 다 넣지 않는다. 저장 스키마를 단계별로 한 번씩만 흔들고, MVP 경계를 단계적으로 해제한다. 각 Phase 끝에서 `SCHEMA_VERSION`/`BALANCE_VERSION`을 한 번 범프하고 회귀를 닫는다.

| Phase | 포함 | 핵심 의존 | 저장 영향 | MVP 해제 |
| --- | --- | --- | --- | --- |
| **P1 (현행)** | 단일 화면·골드·5+4 강화·타입/스킬 | — | v6 (현행) | — |
| **P2 — 표현·UX** | [07-애니메이션](./07-캐릭터-애니메이션.md) 6포즈, [02-파이터](./02-파이터-메인화면.md) 헤더/적카드/진행바, AUTO·배속 | 표현 위주 | 없음~소 | 거의 없음 |
| **P3 — 진행/재화 기반** | [01-공통](./01-공통-레이아웃.md) 다중재화·플레이어레벨·5탭, [05-퀘스트](./05-퀘스트.md) | 재화 모델 = 모든 것의 전제 | 🔴 큼 (1회) | 다중재화·레벨·퀘스트 |
| **P4 — 성장 깊이** | [04-가방/장비](./04-가방-장비.md), [02] 강화 6칸 재구성(장비·훈련·스킬강화) | P3 재화 + 능력치 수식 | 🔴 큼 | 장비·훈련 |
| **P5 — 수익화/경쟁(후순위)** | [03-상점](./03-상점.md, 결제), [06-경기장](./06-경기장-리그.md, PVP) | 결제 심의·서버/더미 | 🔴 큼 | 결제·PVP·랭킹 |

> 권장 순서 = P2 → P3 → P4 → P5. P5(상점·경기장)는 정책·심의·서버 검토가 끝나기 전엔 착수하지 않는다.

1. **재화 모델(P3).** 골드(기존) + **다이아(무과금 획득)** + **플레이어 레벨/경험치** 채택. **에너지 미채택**(도입 시에도 경기장 전용일 것이나 경기장 보류로 현 단계 제외). ⏱타이머는 일일 리셋 카운트다운으로 재해석 — [01-공통](./01-공통-레이아웃.md).
2. **상점/결제.** **무과금 상점만**(골드·다이아). 실결제(IAP)는 앱인토스 v0.1 제외와 정합 → 보류 — [03-상점](./03-상점.md).
3. **강화 패널.** **현행 9종 유지**(6칸 재구성·스킬/장비/훈련강화 축 미채택) — [02-파이터](./02-파이터-메인화면.md).

## 공통 커플링 주의 (모든 파트 공통)

새 재화·아이템·진행 상태는 전부 저장 형식에 닿는다. 어떤 파트든 구현 시:

- `types.ts`(`SaveData`/`Boxer`) → `save.ts` 타입가드 → `gameStore.ts` 직렬화 → `SCHEMA_VERSION` 범프 + 저장 키 접미사(`boxer-game.save.vN`).
- 밸런스 수치(비용·보상·드랍률) → `BALANCE_VERSION` 범프 + 콜로케이트 `*.test.ts`.
- UI·CSS·스토어 변경 → `npm run e2e` 추가 실행.
