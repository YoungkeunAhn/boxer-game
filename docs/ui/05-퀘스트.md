# 퀘스트 (Quests)

출처 이미지: `퀘스트화면.png`. 하단 네비 네 번째 탭.

> **구현 정정 노트(TASK-021):** 본 문서는 퀘스트 저장 상태가 "TASK-019의 v7 범프에 합류"한다고 기술했으나,
> 실제 코드에서 TASK-019(재화·플레이어 레벨)는 v5→**v6** 범프였다(P2가 저장 무변경이라 코드 버전이 v5에 머물렀기 때문).
> 따라서 퀘스트는 questState를 SaveData의 새 top-level 필드로 추가하며 **v6→v7**(SCHEMA 7)을 단독으로 가져간다.
> 밸런스는 퀘스트 보상·리셋 도입으로 **BALANCE 7→8**. 옛 v6/v5 저장은 삭제하지 않고 `legacy`로 안내한다.
> 채택 목표: stageClear/bossClear/killMonster/upgradeStat/autoBattleMinutes/claimFreeChest/playerLevelUp.
> 제외(보류): enhanceEquip·enhanceTraining(장비·훈련), 아이템·에너지 보상. 보상은 골드·다이아만.
> autoBattleMinutes는 온라인 자동 전투 게임시간만 분으로 집계한다(오프라인 제외 — 가정). 주간 리셋은 월요일 00:00(가정).

## 1. 이미지에서 확인된 요소

### 탭
```text
[일일] [주간] [도전] [업적]
```

### 일일 진행도 마일스톤 바
```text
📅 일일 퀘스트 진행도   45 / 100   (갱신까지 07:44:28)
바: ━━━━━━━━━━ 상자 보상 [20][40][60][80][100]  ← 누적 점수 도달 시 상자 수령
```
- 퀘스트 완료마다 점수 적립 → 누적 점수 구간(20/40/60/80/100)에서 보상 상자.

### 퀘스트 리스트 (각 항목)
```text
[아이콘] 제목
        설명 / 진행바 N/M / 보상           [버튼]
```
- 스테이지 3회 클리어 — 2/3 — 🪙5,000 — [이동]
- 훈련 강화 5회 — 5/5 — 🪙7,000 — [수령](완료, 강조 색)
- 장비 강화 3회 — 1/3 — 💎50 — [이동]
- 자동 전투 20분 진행 — 12/20 — 🥊x10 — [이동]
- 일일 무료 상자 받기 — 1/1 — 💎20 — [✓](수령 완료)
- 몬스터 30마리 처치 — 18/30 — 🪙6,000 — [이동]
- 하단: "일일 퀘스트는 매일 00:00에 초기화됩니다."

### 버튼 3상태
- **이동**(미완료, 해당 화면으로 이동) / **수령**(완료, 보상 받기) / **✓**(수령 완료, 비활성).

## 2. 현재 상태

- **퀘스트 시스템 없음.** 일일/주간/도전/업적, 진행 추적, 보상 수령 전무.
- 진행 추적이 필요한 지표 중 일부는 기존 데이터로 파생 가능: 총 처치 수(`boxer.kills`), 스테이지 클리어, 강화 횟수. 단 **카운터 누적·리셋·수령 상태**는 신규 저장 필요.

## 3. 작성 내용 (결정 반영)

> 결정(2026-06-21): **퀘스트 채택(P3)** — 다이아·경험치의 주요 획득원. 단 **장비강화·훈련강화 목표와 아이템/에너지 보상은 보류 시스템이라 제외**(보상은 골드·다이아만). 미확정 수치는 `가정:`/`TODO`.

### 3-1. 퀘스트 데이터 모델
- 정의: id, 카테고리(일일/주간/도전/업적), 목표 타입(아래 — 현재 시스템으로 추적 가능한 것만), 목표 수치, 보상(골드·다이아), 진행 점수(마일스톤 기여).
- 진행 상태 저장: 퀘스트별 current/완료/수령 여부, 일일 누적 점수, 마일스톤 상자 수령 여부, 일일 시작 스냅샷.
- 리셋 규칙: 일일 00:00([01-공통 일일 리셋 타이머](./01-공통-레이아웃.md)와 동일 기준), 주간 월요일 00:00(`가정:`), 도전/업적은 비리셋(영구).

### 3-2. 진행 추적(이벤트 훅)
- 추적 가능한 목표만 채택(보류 시스템 의존 목표 제외):
  - `killMonster` ← `resolveAttack`의 처치 이벤트(`boxer.kills` 증가).
  - `stageClear` / `bossClear` ← 스테이지 전진·보스 클리어.
  - `upgradeStat` ← `purchaseUpgrade` 성공(9종 강화).
  - `autoBattleMinutes` ← `advanceCombat` 누적 경과(주입 `now`). 오프라인 포함 여부 `TODO`(`가정:` 온라인 진행만 집계 권장 — 방치 시간으로 자동 달성 방지).
  - `claimFreeChest` ← [03-상점](./03-상점.md) 무료 상자 수령.
  - `playerLevelUp` ← 플레이어 레벨업([01](./01-공통-레이아웃.md)).
- **제외 목표**: `enhanceEquip`(장비 보류)·`enhanceTraining`(훈련 보류). → 이미지의 "장비 강화 3회"·"훈련 강화 5회" 퀘스트는 해당 시스템 도입(P4) 전까지 미사용.
- 기존 누적값(`boxer.kills`)은 리셋이 안 되므로 **일일 시작 스냅샷** 기준 증분으로 계산(저장에 스냅샷 필요).

### 3-3. 보상/수령
- 보상 종류 = **골드·다이아만**([01-재화](./01-공통-레이아웃.md)와 일치). 아이템(🥊 등)·에너지 보상은 보류 → 골드/다이아로 대체.
- 수령 트랜잭션: 중복 수령 방지(`claimed` 플래그) + 마일스톤 상자 별도 수령(`milestonesClaimed`).
- 하단 네비 알림 뱃지: 완료·미수령 퀘스트 또는 마일스톤 상자 수령 가능 시 표시([01 §3-2](./01-공통-레이아웃.md)).

### 3-4. UI
- [이동] 버튼 라우팅: 스테이지/처치 목표 → 파이터, 강화 목표 → 파이터(강화 패널), 무료 상자 → 상점. (장비→가방 라우팅은 가방 보류로 미사용)
- 마일스톤 진행 바 + 상자 5개(20/40/60/80/100) 표현.

## 3-5. 데이터 모델 초안 (결정 반영)

```ts
// 퀘스트 (P3) — 추적 가능한 목표만, 보상은 골드·다이아
type QuestCategory = 'daily' | 'weekly' | 'challenge' | 'achievement'
type QuestGoalType =
  | 'stageClear' | 'bossClear' | 'killMonster' | 'upgradeStat'
  | 'autoBattleMinutes' | 'claimFreeChest' | 'playerLevelUp'
  // 제외(보류): 'enhanceEquip' | 'enhanceTraining'

type QuestReward = { gold?: number; diamond?: number }   // 아이템·에너지 제외

type QuestDef = {
  id: string
  category: QuestCategory
  goalType: QuestGoalType
  target: number          // 3회, 30마리, 20분…
  reward: QuestReward
  points: number          // 일일 진행도 마일스톤 기여 점수
}

// 저장 상태 (정의와 분리)
type QuestState = {
  progress: Record<string, number>      // questId → current
  claimed: Record<string, boolean>      // 보상 수령 여부
  dailyPoints: number                   // 45 / 100
  milestonesClaimed: number[]           // [20,40] 등 수령한 구간
  dailySnapshot: Record<string, number> // 일일 시작 시 누적값(kills 등) 스냅샷
  resetAt: { daily: number; weekly: number }   // 다음 리셋 epoch
}
```

### 리셋 수식 (`가정:`)
```text
daily resetAt = 다음 로컬 00:00. now >= resetAt 이면 progress/claimed/dailyPoints/dailySnapshot 초기화
주간: 매주 월요일 00:00 (가정)
모두 주입 now 기준 — Date.now 직접 금지
```

## 4. 범위·커플링 플래그

- 🟡 **퀘스트 채택(P3).** [MVP 범위](../overview/mvp-scope.md) "제외"에서 해제 필요. 재화·레벨([01](./01-공통-레이아웃.md))과 같은 P3 범프에 합류.
- 🔴 퀘스트 진행/수령 상태 = 저장 스키마 추가 → `SCHEMA_VERSION`.
- 🟢 장비강화·훈련강화 목표 제외로 [04-가방](./04-가방-장비.md)·훈련 시스템 **전제 없이 독립 도입 가능**.
- 🟡 리셋·시간 추적은 순수 로직(`now` 주입) 필수 — `Date.now` 직접 금지.
