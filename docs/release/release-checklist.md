# 출시 체크리스트

## 문서 목적

복서키우기 자동 전투 MVP의 개발 완료부터 앱인토스 제출 직전까지 필요한 작업을 점검한다. 정책과 SDK 요구사항은 제출 시점에 다시 확인한다.

## 개발 완료

- [ ] 복서 생성 → 자동 공격 → 처치 골드 → 강화 → 보스 → 다음 장 흐름이 연결되어 있다.
- [ ] 보스 실패 후 같은 장 4스테이지를 반복 파밍하고 재도전할 수 있다.
- [ ] 4장 이후 숲 입구·늑대 숲·바위 협곡 테마가 무한 순환한다.
- [ ] 능력치, 피해, 강화 비용, 전이와 오프라인 계산이 UI와 분리되어 있다.
- [ ] 첫 공격 지연, 치명타 경계, 보스 종료 시각의 공격 우선 규칙이 테스트되어 있다.
- [ ] 잘못된 입력이나 저장 데이터에도 화면이 중단되지 않는다.
- [ ] 프로덕션 빌드에 불필요한 디버그 로그가 없다.

## 앱인토스 연동

- [ ] 게임이 토스 앱 안에서 정상 실행된다.
- [x] 앱인토스 WebView SDK 2.9.1과 필수 설정을 확인했다.
- [ ] `ait build`로 최신 `boxer-idle.ait` 번들을 생성했다.
- [ ] 뒤로가기, 백그라운드 전환, 복귀와 종료 흐름을 확인했다.
- [ ] 백그라운드 동안 온라인 전투 타이머가 멈추고 복귀 정산 후 하나만 재시작한다.
- [ ] 실제 기기와 토스 앱 QR 테스트를 완료했다. (`공식 문서 확인 필요`)
- [ ] 허용 도메인, 네트워크와 번들 규칙을 확인했다. (`공식 문서 확인 필요`)

## 저장 및 오프라인 보상

- [ ] `boxer-game.save.v2`, `schemaVersion: 2`, `balanceVersion: 2`를 사용한다.
- [ ] 새로고침 후 복서, 골드, 처치 수, 강화 레벨과 진행 위치가 유지된다.
- [ ] 몬스터 HP와 타이머는 저장되지 않고 안전한 새 전투로 복원된다.
- [ ] 일반 저장은 최대 초당 한 번, 강화·보스 결과·백그라운드·종료는 즉시 반영된다.
- [ ] 오프라인 인정 시간이 최대 8시간이며 같은 기간의 보상이 중복되지 않는다.
- [ ] 보스에서 이탈하면 같은 장 4스테이지를 기준으로 정산한다.
- [ ] v1 데이터는 삭제되지 않고 호환 불가 안내와 새 게임 선택지가 표시된다.
- [ ] 손상 데이터, 저장소 접근 실패와 용량 초과를 처리한다.

## QA

- [ ] 초기 능력치와 5개 강화의 비용·증가량·상한이 기획값과 일치한다.
- [ ] 스테이지 기본 HP·골드와 장별 `1.8`·`1.6` 배율이 일치한다.
- [ ] 일반 처치, 보스 성공·실패와 연속 입력으로 보상이 중복되지 않는다.
- [ ] 공격속도 변경 후 타이머가 중복되지 않고 새 간격이 반영된다.
- [ ] 보스 제한 시각과 같은 시각의 공격을 먼저 처리한다.
- [ ] 360px과 데스크톱에서 긴 이름·큰 숫자·HP 바·강화 버튼이 깨지지 않는다.
- [ ] 신규 생성, 초기화, 새로고침, 백그라운드 복귀를 반복 테스트한다.
- [ ] 지원 대상 OS와 기기 범위를 확인한다. (`공식 문서 확인 필요`)

## 리소스와 등급분류

- [ ] 앱 아이콘, 썸네일, 자동 전투·보스·강화 스크린샷과 설명 문구를 준비했다.
- [ ] 제출 리소스의 크기와 형식이 최신 기준에 맞는다. (`공식 문서 확인 필요`)
- [ ] 사용 권한을 확인한 이미지, 폰트와 사운드만 포함했다.
- [ ] 비현실적 몬스터 타격의 폭력성 표현을 최종 빌드로 검토했다.
- [ ] 결제, 광고, 랜덤박스와 유료 확률형 아이템이 없음을 확인했다.
- [ ] 고객 문의 경로와 개인정보 안내 필요 여부를 확인했다.
