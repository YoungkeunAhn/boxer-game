# 앱인토스 출시 전략

## 문서 목적

복서키우기 자동 전투 MVP를 앱인토스 미니앱으로 출시하기 위한 범위와 검증 방향을 정리한다.

## 출시 플랫폼과 개발 방식

- 토스 앱인토스 WebView 미니앱 출시를 목표로 한다.
- React + Vite + TypeScript의 CSR 정적 빌드로 구성한다.
- 앱인토스 런타임 API는 게임 로직과 분리된 `src/platform` 어댑터로 연동한다.
- 앱인토스 지원 환경, 제출 절차와 검수 기준은 출시 시점의 `공식 문서 확인 필요`.

## SDK 연동 기준선

- 확인일: 2026-06-19
- SDK: `@apps-in-toss/web-framework@2.9.1`
- 설정 파일: `granite.config.ts`
- 앱 유형: `webViewProps.type: "game"`
- 앱 번들 명령: `ait build`
- SDK 3.x는 현재 베타이며 설정 체계가 달라 이번 버전에서는 사용하지 않는다.

가정: 콘솔 등록 전 임시 `appName`은 `boxer-idle`, 대표 색상은 `#f6bb43`을 사용한다.

## 개발 및 빌드 명령

| 명령 | 용도 |
| --- | --- |
| `npm run dev` | Granite 개발 서버와 앱인토스 샌드박스 연동 실행 |
| `npm run build` | 앱인토스 `.ait` 번들 생성 |
| `npm run dev:web` | SDK 브리지 없이 Vite 브라우저 실행 |
| `npm run build:web` | TypeScript 검사 후 일반 웹 빌드 생성 |
| `npm run preview:verify` | 고정 포트 프로덕션 프리뷰 실행 |

## 샌드박스 준비

- Android 에뮬레이터 또는 USB 연결 기기는 `adb reverse tcp:8081 tcp:8081`과 `adb reverse tcp:5173 tcp:5173`을 실행한다.
- iOS 실기기는 개발 PC와 같은 Wi-Fi에 연결하고 설정의 `web.host`를 접근 가능한 LAN IP로 바꾼다.
- 샌드박스 앱에서 콘솔용 토스 비즈니스 개인 계정으로 로그인하고 등록 앱을 선택한다.
- 개발 서버 실행 후 `intoss://boxer-idle` 스킴으로 진입한다.
