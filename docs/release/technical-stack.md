# 기술 스택

## 문서 목적

복서키우기 자동 전투 MVP의 구현 기술과 코드 분리 원칙을 정의한다.

## 기술 스택 요약

| 영역 | 선택 | 용도 |
| --- | --- | --- |
| 빌드 | Vite | 개발 서버와 정적 프로덕션 빌드 |
| UI | React + TypeScript | 전투·강화 화면과 정적 타입 검증 |
| 상태 | Zustand | 게임 상태, 자동 전투와 UI 액션 관리 |
| 스타일 | CSS | 360px WebView 반응형 화면 |
| 저장 | localStorage | v2 진행도와 정산 기준 시각 저장 |
| 테스트 | Vitest | 수식, 전이, 저장과 가짜 타이머 테스트 |
| 플랫폼 | 앱인토스 WebView SDK | 앱 생명주기와 플랫폼 연동 |

## 프론트엔드와 상태 구성

- React 컴포넌트는 상태 표시와 강화·보스 재도전 입력 연결만 담당한다.
- TypeScript `strict` 모드를 사용하고 저장 상태와 전투 런타임 타입을 분리한다.
- Zustand 스토어는 시각, 난수, 스케줄러와 취소 함수를 생성 시 주입받는다.
- 자동 전투는 단일 타이머만 유지하고 실제 시각으로 다음 공격과 보스 남은 시간을 계산한다.
- React StrictMode의 마운트·정리 반복에도 유령 타이머가 남지 않게 한다.

## 저장 방식

- 저장 키는 `boxer-game.save.v2`, `schemaVersion`과 `balanceVersion`은 2다.
- 강화 레벨, 골드, 총 처치 수, 진행 위치와 마지막 저장 시각만 저장한다.
- 몬스터 현재 HP와 공격·보스 타이머는 저장하지 않고 로드 시 재구성한다.
- 일반 저장은 최대 초당 한 번, 강화·보스 결과·백그라운드·종료는 즉시 수행한다.
- 불러올 때 타입, 유한성, 범위와 버전을 검증한다.
- v1 저장은 삭제하지 않고 호환 불가 안내 후 새 v2 게임을 만들 수 있게 한다.

## 앱 생명주기와 오프라인 처리

- `visibilitychange`와 페이지 종료 경로에서 전투를 멈추고 정산 기준을 저장한다.
- 복귀 시 최대 8시간의 현재 일반 스테이지 반복 파밍을 순수 함수로 계산한다.
- 보스에서 이탈했다면 같은 장 4스테이지를 기준으로 정산한다.
- SDK 생명주기 API를 추가할 때는 `src/platform` 어댑터로 격리하고 브라우저 대체 경로를 유지한다.

## 게임 로직 분리 원칙

- 능력치, 강화 비용, 피해, 골드, 스테이지 생성·전이와 오프라인 정산은 UI와 분리한다.
- 정적 테마 정의와 저장되는 진행 상태를 분리한다.
- 시간·난수·저장소 같은 외부 의존성은 함수 인자나 어댑터로 주입한다.
- 스토어가 localStorage를 직접 다루지 않고 저장 모듈 인터페이스를 사용한다.
- 보스 경계 시각, 중복 타이머, 저장 복원과 정산 중복에 자동 테스트를 작성한다.

## v0.1에서 사용하지 않을 기술

- Unity, Godot 등 별도 게임 엔진
- SSR 프레임워크와 서버 렌더링
- 서버 데이터베이스, 계정 동기화, 실시간 통신
- 결제, 광고, 리더보드, PVP용 SDK

## 추후 확장

- 서버 저장 도입 시 저장소 인터페이스의 localStorage 구현을 API 구현으로 교체한다.
- 콘텐츠 데이터는 JSON 또는 관리 도구로 옮길 수 있도록 ID와 장 위치로 참조한다.
- 분석, 원격 설정, 계정 연동은 사용자 동의와 앱인토스 정책을 확인한 뒤 추가한다. (`공식 문서 확인 필요`)
