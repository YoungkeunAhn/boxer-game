# 작업 요청: TASK-007 제출 전 최종 QA 및 반려 리스크 점검

현재 복서키우기 v0.1은 앱인토스 WebView 미니앱 출시를 목표로 개발 중이다.

이번 작업의 목표는 앱인토스 검토 요청 전에 v0.1 게임, 리소스, 문서, 등급분류 자료, WebView 동작을 최종 점검하고 반려 가능성이 높은 항목을 미리 정리하는 것이다.

이번 작업에서는 새로운 게임 기능을 추가하지 않는다.

## 이번 작업 목표

앱인토스 검토 요청 전에 제출 가능한 상태인지 최종 점검한다.

## 점검 범위

1. v0.1 게임 기능 QA
   - 복서 생성
   - 복서 상태 표시
   - 훈련 실행
   - 능력치 증가
   - 전투력 계산
   - 경기 실행
   - 승리/패배 결과 표시
   - 보상 지급
   - localStorage 저장
   - localStorage 불러오기
   - 재접속 시 진행도 유지

2. 모바일 WebView QA
   - 토스 앱 또는 앱인토스 테스트 환경에서 첫 화면 정상 표시
   - 360px 너비에서 UI 깨짐 없음
   - 가로 스크롤 없음
   - 버튼 터치 영역 충분함
   - 하단 버튼이 Safe Area에 가리지 않음
   - 뒤로가기 또는 종료 상황에서 데이터 손실 없음
   - 로딩 중 흰 화면이 오래 지속되지 않음

3. 앱인토스 제출 자료 QA
   - 앱 이름 확인
   - appName 확인
   - 한 줄 소개 확인
   - 짧은 앱 설명 확인
   - 긴 앱 설명 확인
   - 앱 아이콘 준비 여부 확인
   - 대표 썸네일 준비 여부 확인
   - 스크린샷 준비 여부 확인
   - 고객 문의 이메일 준비 여부 확인

4. 게임 등급분류 자료 QA
   - 게임 설명 자료 준비 여부 확인
   - 주요 화면 스크린샷 준비 여부 확인
   - 플레이 영상 준비 여부 확인
   - 기능 목록 정리 여부 확인
   - 폭력성 표현 수위 설명 여부 확인
   - 결제/광고/확률형 아이템 없음 명시 여부 확인
   - 개인정보 수집 여부 정리 여부 확인

5. 반려 리스크 점검
   - 실제 구현되지 않은 기능이 앱 설명에 포함되어 있는지 확인
   - 결제/광고/리더보드/친구 초대 문구가 잘못 들어갔는지 확인
   - 등급분류 관련 자료가 누락되었는지 확인
   - 게임 화면이 토스 앱 UI와 겹치는지 확인
   - 저장 데이터가 사라지는 케이스가 있는지 확인
   - 에러 화면 또는 빈 화면이 발생하는지 확인
   - 정책 확인이 필요한 항목이 TODO로 남아 있는지 확인

## 이번 작업에서 하지 않을 것

- 신규 게임 기능 추가
- 신규 상대 대량 추가
- 장비 시스템 확장
- 코치 시스템 확장
- 인앱 결제 추가
- 광고 추가
- 리더보드 추가
- 친구 초대 추가
- 서버 저장 추가
- 출시 검토 요청

## 수정 원칙

- 출시 직전 단계이므로 기능 추가보다 안정화에 집중한다.
- 실제 구현된 기능만 앱 설명에 남긴다.
- 구현되지 않은 기능은 설명과 스크린샷에서 제거한다.
- 해결이 어려운 항목은 TODO가 아니라 출시 차단 이슈인지 판단한다.
- 출시 차단 이슈와 출시 후 개선 이슈를 분리한다.

## 완료 기준

- v0.1 핵심 게임 루프가 실기기에서 정상 동작한다.
- 저장/불러오기가 재접속 후에도 정상 동작한다.
- 모바일 WebView 화면이 깨지지 않는다.
- 앱 제출 자료가 누락 없이 정리된다.
- 게임 등급분류 자료 준비 상태가 확인된다.
- 반려 가능성이 높은 항목이 정리된다.
- 출시 검토 요청 전 남은 작업 목록이 명확해진다.
