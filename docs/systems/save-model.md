# 저장 모델

저장 형식은 [데이터 모델](./data-model.md)의 타입을 기반으로 한다. 저장 동작 규칙은 [게임 시스템](./game-systems.md)을 함께 본다.

## 저장 타입

```ts
type SaveDataV6 = {
  schemaVersion: 6;
  balanceVersion: 10;
  savedAt: string;
  boxer: Boxer; // boxerType·gender·upgradeLevels(9키)·equippedSkills 포함
  position: StagePosition;
  isFarming: boolean;
};
```

- 저장 키는 `boxer-game.save.v6`다.
- `수정내용2`(TASK-004~012)로 스키마가 단계별로 v2→v6까지 올라갔고(v3 타입·성별, v4 HP·방어 강화, v5 회피·카운터 강화, v6 `equippedSkills`), TASK-013에서 v6을 최종 확정 스키마로 동결했다(재번호 없음).
- 성별은 외형 전용이라 전투 보정이 없다. 밸런스 버전은 타입 보정·HP/방어·회피·카운터·기본 공격·콤비네이션·그로기·스킬 도입으로 누적해 v8이 됐다.
- 날짜는 UTC ISO 8601 문자열로 저장한다.
- 저장에는 정적 테마 데이터, 현재 몬스터 HP, 다음 공격 시각, 보스 종료 시각, 복서 HP·콤보·그로기·스킬 쿨다운/버프 등 런타임 값을 포함하지 않는다.
- 로드 시 위치에서 `StageDefinition`과 새 최대 HP를 생성한다.

## 저장 및 검증 규칙

- `schemaVersion === 6`, `balanceVersion === 10`을 확인한다(둘 중 하나라도 어긋나면 `invalid`).
- 복서 이름은 기존 UI의 길이·공백 검증을 적용한다(1~16자).
- `boxerType`은 `INFIGHTER`/`OUT_BOXER`, `gender`는 `MALE`/`FEMALE` 중 하나여야 한다.
- 골드, 총 처치 수와 강화 레벨(9키: 공격력·공격속도·치명타율·치명타 피해·골드 보너스·체력·방어·회피·카운터)은 음수가 아닌 유한한 안전한 정수여야 하며 각 강화의 레벨 상한을 넘지 않아야 한다.
- `equippedSkills`: 액티브 슬롯은 최대 3개이며 모두 해당 타입(또는 공유)의 액티브 스킬이어야 한다. 교차 타입·패시브를 액티브 슬롯에 넣음·중복은 거부한다. 패시브는 `null` 또는 해당 타입의 패시브 스킬 1개만 허용한다. 형태가 어긋나면 `invalid`.
- 장은 1 이상, 스테이지는 1~5여야 한다.
- 반복 파밍 상태는 보스가 아닌 위치(4스테이지)에만 저장하며, 정상 전이는 같은 장 4스테이지를 사용한다.
- 지원하지 않는 구버전 키(`...v5`~`...v1`)는 누락 필드(강화·`equippedSkills` 등)가 있어 자동 마이그레이션할 수 없다. 삭제·덮어쓰지 않고 호환 불가(legacy) 안내에만 사용한다. (가정: 마이그레이션을 택한다면 누락 필드를 기본값으로 부여해 v6로 승격할 수 있으나, 개발 중 스키마 변동이 잦아 새 게임 진입을 기본으로 둔다.)
- 저장소 접근·파싱·직렬화 오류는 화면을 중단시키지 않고 실패 상태로 반환한다.
- 일반 변경은 최대 초당 한 번 저장하고 강화·보스 결과·백그라운드·종료는 즉시 저장한다.

## 복원과 오프라인 정산

- 저장 시각부터 현재 시각까지를 0~8시간으로 제한해 한 번 정산한다.
- 저장 위치가 보스면 같은 장 4스테이지의 반복 파밍 상태로 보정한다.
- 일반 위치면 해당 위치를 유지한 채 같은 몬스터만 반복 처치한 것으로 계산한다.
- 정산 후 `savedAt`을 즉시 갱신해 중복 보상을 막는다.
- 복원된 온라인 전투는 몬스터 최대 HP와 새 공격 간격으로 시작한다.
- 오프라인 보상 수식은 [능력치와 수식](./stats-and-formulas.md)을 따른다.

## 수정내용2 확장 (구현 완료)

`수정내용2`로 추가된 저장 필드는 모두 도입 완료다: 복서 타입·성별(v3), HP·방어 강화 레벨(v4), 회피·카운터 강화 레벨(v5), 장착 스킬 `equippedSkills`(v6). 복서 HP·콤보·그로기·스킬 쿨다운/버프/내상 DoT는 `CombatRuntime` 런타임 전용이라 저장하지 않는다(스키마 영향 없음). 형식을 바꾸면 본 문서와 [데이터 모델](./data-model.md)을 함께 갱신한다.
