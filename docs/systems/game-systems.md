# 게임 시스템

가정: 본 문서의 「복서와 능력치」·「자동 전투」는 현재 구현 기준선(5종 능력치, 몬스터 무공격)이다. `수정내용2`가 더하는 복서 타입·HP·몬스터 공격은 하단 「수정내용2 신규 시스템」과 [몬스터 공격](../combat/monster-attacks.md)에서 다룬다.

## 복서와 능력치

- 복서는 이름, 골드, 총 처치 수와 5개 강화 레벨을 보유한다.
- 전투 능력치는 `attackPower`, `attackSpeed`, `critRate`, `critDamage`, `goldBonus`다.
- 현재 구현 기준선에서 체력·방어력·회피·카운터는 사용하지 않는다(수정내용2에서 도입).
- 강화 레벨만 저장하고 표시 능력치는 매번 순수 함수로 계산한다.

## 자동 전투

- 공격속도에 따라 자동 공격하고, 치명타율과 주입된 난수로 치명타를 판정한다.
- 현재 구현 기준선에서 몬스터는 복서를 공격하지 않는다(수정내용2에서 몬스터 공격 도입).
- 타이머 드리프트와 React StrictMode 중복 실행을 막기 위해 전투 타이머는 하나만 유지한다.
- 공격속도가 바뀌면 실제 현재 시각을 기준으로 다음 공격 일정을 다시 구성한다.

## 스테이지와 보스

- 한 장은 1~4 일반 스테이지와 5 보스 스테이지로 구성된다.
- 일반 처치 시 다음 번호로 자동 전진하고 보스 처치 시 다음 장 1로 이동한다.
- 보스 제한 시간은 30초다. 실패하면 같은 장 4에서 일반 몬스터를 반복 생성한다.
- 반복 파밍 상태에서 사용자가 재도전하면 진행 중인 일반 전투를 취소하고 보스를 처음부터 시작한다.
- 3개 테마를 장 번호로 순환하며 장 수에는 상한이 없다.

## 강화

- 공격력, 공격속도, 치명타율, 치명타 피해, 골드 보너스를 각각 골드로 강화한다.
- 강화는 잔액과 상한을 먼저 확인한 뒤 비용 차감과 레벨 증가를 한 트랜잭션으로 처리한다.
- 강화 직후 즉시 저장하며, 비용과 수식은 [능력치와 수식](./stats-and-formulas.md)을 따른다.

## 오프라인 보상

- 백그라운드 진입과 페이지 종료 시 정산 기준 시각을 저장하고 온라인 전투를 중지한다.
- 복귀·재접속 시 경과 시간을 0~8시간으로 제한한다. 시스템 시각이 과거면 0초로 처리한다.
- 현재 일반 스테이지의 HP와 골드로 반복 파밍을 계산하며 스테이지는 이동하지 않는다.
- 보스 중 이탈한 경우 직전 일반 스테이지를 정산 대상으로 사용한다.
- 정산 기준 시각을 즉시 갱신해 같은 기간의 보상을 두 번 지급하지 않는다.

## 저장

- 저장 키는 `boxer-game.save.v2`, 스키마와 밸런스 버전은 모두 2다.
- 몬스터 현재 HP, 다음 공격 시각, 보스 종료 시각 같은 런타임 값은 저장하지 않는다.
- 일반 변경의 저장 호출은 최대 초당 한 번으로 제한한다.
- 강화, 보스 결과, 백그라운드 진입과 페이지 종료는 즉시 저장한다.
- v1 데이터는 삭제하지 않고 호환 불가를 안내하며 새 게임 생성 선택지를 제공한다.

## 공통 구현 원칙

- 정적 스테이지 정의와 플레이어 진행 상태를 분리한다.
- 시각, 난수, 스케줄러와 취소 함수는 스토어 생성 시 주입한다.
- 피해, 처치, 보상, 전이, 강화와 오프라인 정산은 가능한 한 순수 함수로 구현한다.
- 모든 값은 유한성과 범위를 검증하고 증가 수치는 안전한 정수 범위로 제한한다.

## 수정내용2 신규 시스템

가정: 아래는 `수정내용2` 설계 방향이며 밸런스·저장 형식은 미확정이다. 도입 시 본 문서·[데이터 모델](./data-model.md)·밸런스 버전을 함께 갱신한다.

- 복서 타입(인파이터/아웃복서)과 성별 외형 → [복서 타입](../boxer/types.md)
- 복서 HP·몬스터 공격·회피/가드/카운터 → [몬스터 공격](../combat/monster-attacks.md), [체력과 실패](../combat/hp-and-defeat.md)
- 기본 공격 4종·콤비네이션·그로기 → [공통 기본 공격](../combat/basic-attacks.md), [보스전](../combat/boss.md)
- 타입 전용 스킬과 장착 슬롯 → [스킬 장착 구조](../skills/equip.md)
