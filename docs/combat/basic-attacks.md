# 공통 기본 공격

모든 타입은 공통 기본 공격 4개를 가진다. 4개는 쿨타임 기반 자동 공격으로 동작하며 항상 사용된다. → [스킬 장착 구조](../skills/equip.md)

```text
잽 / 스트레이트 / 훅 / 어퍼
```

아래 쿨타임·데미지 등급은 `수정내용2`로 구현 완료된 확정값이다(현재 balanceVersion 10; 수치 자체는 `수정내용2` 마감값에서 변하지 않았고, 강화 시 쿨타임 진척 보존 버그 수정으로 9→10 범프).

## 공격별 규칙

| 공격 | 손 | 쿨타임 | 데미지 | 역할 |
| --- | --- | ---: | --- | --- |
| 잽 | 왼손 고정 | 1초 | 낮음 | 기본 딜, 콤보 시작 |
| 스트레이트 | 오른손 고정 | 5초 | 중~높음 | 원투 마무리, 카운터 핵심 |
| 훅 | 왼손/오른손 | 10초 | 높음 | 치명타, 그로기 누적 |
| 어퍼 | 왼손/오른손 | 15초 | 매우 높음 | 보스 그로기, 마무리 |

## 강화 항목

```text
잽:       데미지↑, 쿨타임↓, 2연타 확률↑, 콤보 게이지 증가량↑
스트레이트: 데미지↑, 쿨타임↓, 원투 보너스↑, 카운터 데미지↑
훅:       데미지↑, 치명타 확률↑, 쿨타임↓, 보스 그로기↑
어퍼:     데미지↑, 쿨타임↓, 그로기 증가량↑, 그로기 적 추가 데미지↑
```

## 강화 시 공격 쿨타임 동작 (진척 보존)

스탯 강화로 공격 속도가 바뀔 때, 진행 중이던 공격 쿨타임의 **진척(progress)을 보존**해 재스케줄한다(`rescheduleAttacks`). 전체 리셋이 아니다.

```text
타입별 remaining = max(0, nextReadyAt[type] - now)
fraction = oldCd > 0 ? min(1, remaining / oldCd) : 0
새 nextReadyAt[type] = now + fraction × newCd   (oldCd/newCd = 강화 전/후 실효 쿨타임)
```

- 같은 속도(또는 미세 변화)로 강화를 **연타해도** 다음 공격 시각이 사실상 유지되어 공격이 끊기지 않는다.
- 공격 속도가 오르면 다음 공격이 당겨지거나 같으며, **절대 더 멀어지지 않는다**.
- 과거 버그: 강화 때마다 쿨타임을 `now + 쿨타임`으로 전체 리셋해, 누적 골드를 쓰려 강화를 연타하면 `nextAttackAt`이 계속 미래로 밀려 "강화하면 공격이 멈추는" 증상이 있었다. → 진척 보존으로 수정(balanceVersion 9→10, 저장 형태 불변이라 SCHEMA 6 유지).

## 자연스러운 복싱 동작 규칙

기본 자세는 오소독스 기준이다. 어느 손으로 치는지까지 관리해야 자연스럽다.

```text
잽 = 왼손 고정
스트레이트 = 오른손 고정
훅/어퍼 = 손 선택 가능
```

손 선택 규칙(훅·어퍼):

```text
직전 공격이 왼손이면 오른손 우선
직전 공격이 오른손이면 왼손 우선
콤보에 손이 지정되어 있으면 지정 손 우선
```

자연스러운 콤보 예시:

```text
잽 → 스트레이트          = left_jab → right_straight
잽 → 스트레이트 → 훅       = left_jab → right_straight → left_hook
잽 → 스트레이트 → 훅 → 어퍼 = left_jab → right_straight → left_hook → right_upper
```

## 구현 확정값 (TASK-007, `src/game/constants.ts`)

아래 값은 TASK-013에서 확정·동결했다(현재 balanceVersion 10; 아래 수치는 변동 없음). 코드·테스트와 같은 값을 유지한다.

- 쿨타임(공격 속도 1.0 기준): 잽 1초 / 스트레이트 5초 / 훅 10초 / 어퍼 15초. 실효 쿨타임 = 쿨타임 / `attackSpeed`.
- 데미지 계수(`attackPower` 배수): 잽 0.3 / 스트레이트 1.5 / 훅 2.0 / 어퍼 3.0.
  - 초당 가중합 Σ(계수 / 쿨타임초) = 0.3 + 1.5/5 + 2.0/10 + 3.0/15 = **1.0** 으로 맞춰, 평균 DPS를 기존 단일 공격(공격력×attackSpeed/초)과 동일하게 유지한다 → 처치·골드·보스 진행·오프라인 정산이 기존과 동일.
- 발동 정책: 매 틱 쿨타임이 끝난(ready) 공격 중 우선순위 **어퍼 > 훅 > 스트레이트 > 잽** 으로 한 종류를 친다. 쿨타임이 짧은 잽이 가장 자주, 어퍼가 가장 드물게 나간다.
- 손 선택: 잽=왼손·스트레이트=오른손 고정, 훅·어퍼는 직전 타격 손과 반대 손(직전 정보 없으면 왼손).
- 애니메이션 키 규약: `boxer_{left|right}_{jab|straight|hook|upper}` (`attackAnimationKey`). TASK-012에서 적용 완료.
- 도입 완료: 콤비네이션 보너스(원투/원투 훅/풀콤보, TASK-008)와 보스 그로기(TASK-009)는 구현 완료다. → [콤비네이션](./combinations.md), [보스전](./boss.md). 공격별 개별 강화는 MVP 밖(미도입).
