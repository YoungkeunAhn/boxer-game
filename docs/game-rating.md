# 게임 등급분류 준비

## 문서 목적

복서키우기 v0.1의 콘텐츠를 출시 전에 자체 점검하고, 실제 등급분류 또는 심의에 필요한 자료와 확인 과제를 준비한다. 이 문서는 법률 자문이나 등급 판정을 대신하지 않는다.

## 검토가 필요한 이유

- 복서키우기는 훈련과 복싱 경기를 핵심으로 하는 게임이므로 게임물 분류 및 폭력성 표현 검토가 필요하다.
- 앱인토스 입점 정책과 유통 경로에 따라 필요한 절차가 달라질 수 있다. (`공식 문서 확인 필요`)
- 기능이나 표현이 바뀌면 기존 판단과 등급이 달라질 수 있으므로 제출 직전에 다시 점검한다.

## 확인할 등급분류 경로

- 게임물관리위원회 등급분류 대상 및 신청 절차 (`공식 문서 확인 필요`)
- 앱스토어 자체등급분류 적용 가능 여부와 조건 (`공식 문서 확인 필요`)
- 플레이스토어 자체등급분류 적용 가능 여부와 조건 (`공식 문서 확인 필요`)
- 원스토어 자체등급분류 적용 가능 여부와 조건 (`공식 문서 확인 필요`)
- 앱인토스가 요구하는 등급 정보, 증빙, 표시 방식 (`공식 문서 확인 필요`)

## 콘텐츠 리스크 점검

| 항목 | v0.1 방향 | 확인 사항 |
| --- | --- | --- |
| 복싱 경기 | 스포츠 경기 중심 | 타격 표현과 반복 노출 수위 |
| 폭력 표현 | 비현실적이고 절제된 연출 | 출혈·부상·잔혹 표현 미포함 |
| 보상 | 경기 결과에 따른 고정 보상 | 우연성과 사행성 오인 요소 배제 |
| 결제·광고 | 미포함 | 관련 버튼, 문구, SDK도 제외 |
| 이용자 데이터 | 기기 로컬 저장 | 수집·전송 여부와 고지 필요성 |

## 폭력성 관련 검토

- 복싱은 격투 스포츠이므로 타격 애니메이션, 효과음, 화면 흔들림의 수위를 낮춘다.
- 캐릭터 타격을 과격하거나 현실적으로 묘사하지 않는다.
- 피격에 따른 출혈, 상처, 골절, 신체 훼손, 사망 표현은 제외한다.
- 부상을 조롱하거나 반복 타격을 보상하는 문구와 연출은 사용하지 않는다.
- 승패와 성장을 강조하고 고통이나 잔혹함을 핵심 재미로 만들지 않는다.

## 결제·광고·랜덤 보상

- v0.1에는 결제, 광고, 랜덤박스, 유료 확률형 아이템을 포함하지 않는다.
- 경기 보상은 규칙을 확인할 수 있는 고정 또는 예측 가능한 방식으로 설계한다.
- 무료 보상이라도 확률을 사용한다면 획득 규칙과 표시 필요 여부를 검토한다. (`공식 문서 확인 필요`)
- 추후 결제나 광고를 추가할 때 등급, 고지, 청소년 보호, 환불 관련 요건을 다시 검토한다. (`공식 문서 확인 필요`)

## 개인정보 및 계정 저장

- v0.1은 localStorage에 게임 진행도와 설정만 저장하고 불필요한 개인정보를 수집하지 않는다.
- 앱인토스 사용자 식별자, 계정 연동, 서버 저장은 v0.1에서 제외한다.
- SDK가 제공하거나 전송하는 식별 정보와 로그 범위를 확인한다. (`공식 문서 확인 필요`)
- 문의 과정에서 이메일 등 개인정보를 받을 경우 보관·파기·고지 방식을 확인한다. (`공식 문서 확인 필요`)

## v0.1 콘텐츠 원칙

- 밝고 비현실적인 그래픽과 절제된 타격 효과를 사용한다.
- 출혈, 부상, 잔혹 표현과 도박을 연상시키는 연출을 제외한다.
- 실제 현금 가치로 교환 가능한 재화나 아이템을 만들지 않는다.
- 연령에 부적절할 수 있는 욕설, 음주, 흡연, 선정적 표현을 제외한다.
- 최종 안전성 판단은 실제 빌드 전체를 기준으로 다시 수행한다.

## 준비 자료

- 게임 소개, 핵심 루프, 전체 기능 및 제외 기능 목록
- 시작부터 경기와 보상 획득까지의 플레이 영상
- 주요 화면, 경기 연출, 보상 화면 스크린샷
- 캐릭터, 대사, 효과음, 타격 및 패배 연출 목록
- 결제·광고·확률형 요소 미포함 확인서 또는 기능 설명
- 저장 데이터 항목과 외부 전송 여부 설명
- 제출 버전과 자료 버전의 일치 여부 기록
