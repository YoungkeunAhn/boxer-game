# 개발 로드맵

## 원칙

- 각 단계는 실행 가능하고 자동 테스트 가능한 게임 상태로 끝낸다.
- 게임 수식과 상태 전이는 UI에서 분리하고 외부 시각·난수·스케줄러를 주입한다.
- 저장 스키마와 밸런스 버전은 규칙 변경과 함께 관리한다.

## v0.1 — 자동 전투 코어

목표: 복서 생성부터 일반 몬스터 자동 처치까지 동작한다.

1. `CombatStats`, `UpgradeLevels`, 스테이지와 저장 v2 타입 정의
2. 3개 테마 순환 스테이지 생성기 구현
3. 능력치, 피해, 골드와 강화 비용 순수 함수 구현
4. 단일 타이머 자동 공격과 일반 스테이지 전이 구현
5. 순수 함수와 타이머 경계 단위 테스트 작성

완료 조건: 사용자의 공격 클릭 없이 1-1부터 1-5까지 자동 진행한다.

## v0.2 — 보스와 강화

목표: 골드 성장과 보스 돌파 루프를 완성한다.

1. 30초 보스 타이머와 경계 시각 공격 우선 처리
2. 보스 성공 시 다음 장, 실패 시 직전 일반 스테이지 전이
3. 반복 파밍과 수동 재도전 구현
4. 5개 골드 강화와 상한·비용 처리
5. 4장 이후 테마 순환과 큰 수 안전성 테스트

완료 조건: 실패 후 파밍·강화·재도전으로 다음 장에 진입할 수 있다.

## v0.3 — 저장과 방치

목표: 앱 생명주기와 재접속에서도 진행을 안전하게 유지한다.

1. 저장·검증·복원 구현(MVP 당시 키 `boxer-game.save.v2`; `수정내용2` 마감 후 최종 키 `boxer-game.save.v6`, `schemaVersion: 6`·`balanceVersion: 9`)
2. 일반 저장 제한과 중요 사건 즉시 저장 구현
3. 최대 8시간 오프라인 반복 파밍 정산 구현
4. 보스 중 이탈과 시스템 시각 역행 처리
5. v1 호환 불가 안내와 손상 저장 오류 흐름 구현

완료 조건: 같은 오프라인 기간의 보상이 중복되지 않고 진행 위치가 안전하게 복원된다.

## v1.0 — MVP 출시 후보

목표: 앱인토스 WebView에서 전체 자동 성장 루프를 안정적으로 반복한다.

1. 자동 전투·강화·오프라인 보상 UI와 접근성 보완
2. 360px, 데스크톱과 백그라운드 수동 스모크 테스트
3. 성능, 저장 안정성, 무한 장 수치 검증
4. 앱인토스 번들·샌드박스·실기기 검증
5. 등급분류와 제출 리소스 준비

완료 조건: 생성, 자동 전투, 강화, 보스 실패·재도전, 재접속 복구가 치명적 오류 없이 동작한다.

## 수정내용2 확장 (구현 완료, TASK-004~013)

`수정내용2`는 TASK-004~012로 구현 완료됐고 TASK-013에서 저장 스키마·밸런스 버전·문서·테스트를 `schemaVersion: 6`·`balanceVersion: 9`로 확정·동결했다.

- v1.1 복서 타입·성별: 인파이터/아웃복서 선택과 외형(완료). → [복서 타입](../boxer/types.md)
- v1.2 전투 심화: 복서 HP·몬스터 공격·회피/가드/카운터(완료). → [몬스터 공격](../combat/monster-attacks.md), [체력과 실패](../combat/hp-and-defeat.md)
- v1.3 스킬: 기본 공격 쿨타임·콤비네이션·그로기·전용 스킬 슬롯(완료). → [스킬 장착 구조](../skills/equip.md)
- v1.4 연출: 게임 화면형 UI·타입별 톤·애니메이션(완료). → [UI 구조](../presentation/ui.md)
- 이후: 장비, 프레스티지, 스토리, PVP, 길드, 랭킹, 시즌, 광고, 결제는 별도 기획 후 검토(MVP 밖)

## 주요 위험

- 타이머 중복: 단일 타이머와 취소 가능한 주입 스케줄러로 방지한다.
- 보상 중복: 처치·보스·오프라인 정산을 원자적으로 처리하고 즉시 저장한다.
- 성장 정체: 보스 도달 시간과 강화 효율을 측정해 버전 단위로 조정한다.
- 시각 조작: 오프라인 경과 시간 상한과 비정상 시각 검사를 적용한다.
- 큰 수 오버플로: 모든 지수 계산을 안전한 정수 범위로 제한한다.
