# 개발 순서

1. 앱인토스 개발자센터와 최신 정책을 확인한다.
2. 자동 전투 능력치, 스테이지, 강화, 저장 v2 타입을 정의한다.
3. 능력치·피해·보상·강화 비용·스테이지 생성 순수 함수를 구현한다.
4. 단일 타이머 자동 공격과 30초 보스 제한 시간을 구현한다.
5. 보스 실패 후 직전 일반 스테이지 반복 파밍과 수동 재도전을 구현한다.
6. 저장/불러오기와 구버전 안내를 구현한다(MVP 당시 키 `boxer-game.save.v2`; `수정내용2` 마감 후 최종 키는 `boxer-game.save.v6`).
7. 최대 8시간 오프라인 반복 파밍 정산을 구현한다.
8. 자동 전투·강화 패널과 360px WebView UI를 구현한다.
9. 단위·스토어 통합 테스트와 브라우저 스모크 테스트를 수행한다.
10. 앱인토스 `.ait` 번들과 실기기 샌드박스를 검증한다.
11. 게임 등급분류, 로고, 썸네일, 설명과 검수 자료를 준비한다.
12. 검토 요청 후 승인된 빌드를 출시한다.
