# AGENTS.md

## 프로젝트 개요

- 프로젝트명은 `복서키우기`다.
- 무명 복서를 훈련해 상위 리그와 챔피언에 도전하는 방치형 복싱 육성 게임이다.
- 핵심 루프는 `훈련 → 능력치 상승 → 경기 → 보상 획득 → 업그레이드 → 더 강한 상대 도전`이다.
- 초기 버전은 React, Vite, TypeScript 기반의 앱인토스 WebView 미니앱을 목표로 한다.

## 파일 읽기 규칙

- 저장소 안의 파일 내용을 읽을 때는 반드시 프로젝트 루트의 `tools/read-md.py`를 사용한다.
- 기본 명령은 `python tools/read-md.py "<파일 경로>"`다.
- 경로에 공백이나 한글이 포함될 수 있으므로 파일 경로는 큰따옴표로 감싼다.
- 인코딩 확인이 필요하면 `python tools/read-md.py --show-encoding "<파일 경로>"`를 사용한다.
- 파일 목록이나 경로 검색에는 `rg --files`를 사용할 수 있지만, 검색한 파일의 내용은 `tools/read-md.py`로 읽는다.
- `Get-Content`, `cat`, `type` 등 다른 명령으로 파일 내용을 직접 읽지 않는다.

## 문서 기준

- 게임 기획의 시작점은 `docs/기획/README.md`다.
- 구현 전에는 관련 기획 문서와 `docs/technical-stack.md`를 확인한다.
- 아직 확정되지 않은 사항은 `TODO`, 검증 전 설계 판단은 `가정:`으로 표시한다.
- 게임 규칙이나 수식이 바뀌면 관련 문서와 데이터 버전을 함께 갱신한다.
- 문서는 특별한 이유가 없다면 한국어와 Markdown 형식으로 작성한다.

## 구현 원칙

- MVP 범위를 우선하며 PVP, 길드, 시즌패스, 결제, 광고 등 제외 기능을 임의로 추가하지 않는다.
- 게임 로직은 UI와 분리한다.
- 전투력 계산, 훈련 처리, 경기 결과 계산은 가능한 한 순수 함수로 작성한다.
- 저장 및 불러오기 로직은 별도 모듈로 분리하고 저장 데이터의 버전을 관리한다.
- 앱인토스 정책이나 SDK 요구사항처럼 변경 가능한 외부 기준은 공식 문서를 확인하고 단정적으로 기록하지 않는다.

## 빠른 검증 원칙

- 구현 중에는 `node tools/check.mjs fast "<변경 파일>"`로 변경 영역과 직접 관련된 검사만 실행한다.
- 여러 파일을 바꿨으면 변경한 경로를 모두 인자로 전달한다.
- 설정, 의존성, 번들 구성 변경이나 최종 릴리스 검증에는 `node tools/check.mjs full`을 실행한다.
- 실행할 검사만 미리 확인할 때는 `node tools/check.mjs plan "<변경 파일>"`을 사용한다.
- 코드 변경 없이 동일한 검증 명령을 반복하지 않는다.
- 서브에이전트 병렬 작업을 사용하라는 요청이 있을 때 검증은 `.codex/agents/fast-verifier.toml`의 전용 에이전트에 위임할 수 있다.
- 단순한 한 파일 변경에서는 서브에이전트 시작 비용이 더 크므로 메인 에이전트가 빠른 검증을 직접 수행한다.
